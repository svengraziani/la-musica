# P28 Time-stretch, pitch-shift, and warp-marker DSP in the render and bounce path

> **Area:** audio-engine · **Severity:** blocker · **Effort:** L · **Phase:** Phase 2 — Generic session authoring + faithful render (audio-engine extension, slots between P09 and P21/P26)
> **Depends on:** P05, P09, P10 · **Blocks:** P21, P26

## Objective
Deliver the real warp/time-stretch/pitch-shift DSP the mission lists as non-negotiable and that P21 (clip-warp UI) and P26 (spectral-correctness tests) already assume exists. Today only planning math exists: `Warp.hpp` defines `WarpState`/`WarpMarker`/`StretchQuality`/`WarpRenderPlan`/`RenderCacheEntry`, and the only "renderer" (`renderWarpedAudio`) is a single rate-change resampler that *couples* pitch and time. Produce a production DSP stage in the Sample-clip render path that (a) resamples asset-rate to engine-rate, (b) performs time-stretch independent of pitch and pitch-shift independent of duration honoring `WarpState`/`WarpMarker` with selectable `StretchQuality` and transient preservation at markers, (c) is deterministic with an edit-invalidated render cache, and (d) is allocation/lock-free on the realtime callback while being bit-comparable offline (bounce) vs live within tolerance.

## Dependencies
- P05 — Validated audio-graph snapshot, precompiled noexcept render path, and true lock-free SPSC command queue
- P09 — Faithful GraphCompiler: render real user audio/MIDI content and add an instrument/synth graph node kind
- P10 — Streaming WAV read/write, incremental recording-to-disk, and accurate sample-rate header handling

## Deliverables
- **New DSP module** `libs/session/src/WarpDsp.cpp` + `libs/session/include/lamusica/session/WarpDsp.hpp` (or `libs/audio/.../WarpDsp.*` if it must link with no `session` dependency — decide below). Implements the actual stretcher: a WSOLA/overlap-add or phase-vocoder grain engine that maps the source timeline through `mapSourceToTimeline` and the warp markers, applies an independent pitch ratio via resampling of stretched grains, and preserves transients at `WarpMarker` boundaries.
- **Replace the coupled resampler** in `libs/session/src/Warp.cpp:350` (`renderWarpedAudio`). Its current core, `sourceOffset = (frame / stretchRatio) * pitchRatio` (`Warp.cpp:373-374`), advances a single read head so a +12 st pitch change *also* halves duration — wrong. Keep the signature/`WarpRenderPlan` contract but route through the new grain engine so stretch and pitch are orthogonal. `renderWarpPreview` (`Warp.cpp:389`) must call the same DSP kernel as offline, not a forked path.
- **Fractional resampler + asset-rate→engine-rate conversion** in the Sample-clip render path. `GraphCompiler.cpp:206-210` currently loads `wav.audio.interleavedSamples` 1:1 with no rate info; `sampleNodeValue` (`AudioGraph.cpp:67`) and `engineSampleNodeValue` (`AudioEngine.cpp:87`) read `node.samples[sourceFrame]` with no resampling. Add an asset sample-rate field to `GraphNode` (`libs/audio/include/lamusica/audio/AudioGraph.hpp:27`) and a polyphase/sinc resampler so a 44.1k asset plays correctly through a 48k engine. (Import-time resampling/reconciliation is P29's job; this task does the *render-path* conversion when an asset arrives at engine rate mismatched.)
- **Live render-path warp**: feed warped/stretched samples into the callback. Per RT rules the grain engine cannot run on the callback; precompute the warped clip buffer off-thread into the `RenderCacheEntry`-backed cache and swap it into the audio graph snapshot (the P05 validated-snapshot mechanism, `AudioEngine.hpp:74` `RealtimeCommandQueue`), so `engineSampleNodeValue` reads an already-warped buffer (or a preallocated grain/ring buffer) with no DSP on the callback.
- **Deterministic render cache**: wire `makeWarpCacheKey` (`Warp.cpp:296`), `findValidRenderCacheEntry` (`Warp.cpp:311`), `upsertRenderCacheEntry`/`invalidateRenderCache` (`Warp.hpp:115-116`) into the GraphCompiler/engine so a warp edit (any field folded into the key at `Warp.cpp:298-303`) invalidates and triggers re-render; identical key ⇒ byte-identical output. Persist cache entries via the P08 sidecar/cache format and the streaming WAV writer from P10.
- **Tests** in the migrated framework (P02): in `tests/unit/` (e.g. `WarpDspTests.cpp`) — FFT asserts pitch-shift +12 st moves spectral peaks one octave with unchanged frame count; 2x stretch doubles frame count with unchanged dominant bin; transient-at-marker assertion; 44.1k→48k SR-conversion pitch/tempo correctness; offline `renderWarpedAudio` vs live callback output agree within tolerance (extend the existing checks around `bootstrap_tests.cpp:5358-5410`).
- **RT-safety audit**: run the warp swap path under the P06 `RealtimeCallbackAudit` trap (`libs/session/include/lamusica/session/Performance.hpp:55`) to prove no allocation/lock on the callback.

## Decisions To Make
- **Stretch algorithm + licensing.** WSOLA/phase-vocoder written in-house (BSD-clean, ships in an open-source DAW) vs. a bundled stretcher. SoundTouch is LGPL (dynamic-link OK), Rubber Band is GPL/commercial-dual — GPL conflicts with a permissively-licensed distribution and a commercial license costs money. Decide before writing the kernel; this gates the whole module and the P27 SBOM/notarization story.
- **`StretchQuality` mapping.** Define what `Preview`/`Balanced`/`High` (`Warp.hpp:13-17`) mean concretely: Preview = cheap linear/grain, Balanced = WSOLA, High = phase-vocoder with phase-locking. Currently `interpolatedSample` (`Warp.cpp:41`) only varies interpolation order, not the stretch engine.
- **Module placement / dependency direction.** WarpDsp in `libs/session` keeps it near `WarpState`, but the live engine in `libs/audio` must not depend on `libs/session`. Decide whether DSP lives in `libs/audio` (engine-facing, session passes plain buffers) or stays in `session` and only the precomputed buffer crosses into the graph snapshot.
- **SR-conversion policy at render time.** Resample-on-load into `node.samples` (simple, larger memory, loses original) vs. keep asset rate on the node and resample inside the read head (more compute, RT-incompatible unless precached). Must align with P29's import-time policy and P10's accurate SR headers.
- **Cache granularity & invalidation scope.** Per-clip full-buffer cache vs. per-grain. Define exactly which `WarpState` edits bust the key (already folded: enabled, tempos, semitones, quality, all markers — `Warp.cpp:298-303`) and whether asset-content changes (re-record/re-import) must also invalidate.
- **Determinism contract.** Fix denormal handling, FFT/window choice, and float accumulation order so the cache key implies bit-identical output across runs and machines (required by P26 determinism gate).

## Compromises And Tradeoffs
- **Accepted:** live playback reads a *precomputed* warped buffer rather than streaming the stretcher on the callback. This trades memory/latency-to-first-warp for hard RT-safety. Revisit if P21 needs sub-buffer-latency live warp dragging (then a bounded RT-safe grain player becomes necessary).
- **Accepted:** transient preservation is marker-driven (snap grain boundaries to `WarpMarker.sourceSample`), not fully automatic onset detection on the audio thread; `detectTransients` (`Warp.hpp:88`) runs off-thread at edit time.
- **Rejected MVP shortcut:** keeping the coupled single-read-head resampler (`Warp.cpp:373`) and pretending pitch is independent — this is exactly the "tests nonexistent behavior" gap P26 must catch; it is removed, not papered over.
- **Rejected:** linking a GPL stretcher into a permissive distribution; rejected unless the project relicenses or dynamically links an LGPL alternative.
- **Risks to revisit:** SR-conversion here only covers the render path; full import reconciliation is P29 (do not duplicate). Cache persistence depends on P08/P10 landing the sidecar+streaming-WAV format; if those slip, ship an in-memory cache and mark the on-disk cache a follow-up. PDC interaction with warp-induced latency is P14's concern — leave a hook, don't solve it here.

## Acceptance Gates
- `cmake --build build && ctest --test-dir build` is green, including the new `WarpDsp` FFT tests.
- **Pitch independence:** a clip pitched +12 st has its dominant FFT bin doubled in frequency while frame count is unchanged (assert within a few-cent tolerance); −12 st halves it.
- **Time independence:** a 2x time-stretch doubles output frame count while the dominant FFT bin frequency is unchanged within tolerance.
- **Transient-at-marker:** with a `WarpMarker` mapping `sourceSample`→`timelineSample`, the rendered transient envelope peak lands at `timelineSample` (within stretch-quality tolerance).
- **SR conversion:** a 44.1k asset rendered through a 48k engine plays at correct pitch and tempo (compare against a reference 48k render).
- **Offline == live:** `renderWarpedAudio(...)` output and the live callback-fed warped buffer agree within sample tolerance via `warpRenderPlansAgree` (`Warp.cpp:394`) *and* a direct sample-diff; identical `makeWarpCacheKey` ⇒ byte-identical buffers across two runs (determinism).
- **Cache invalidation:** editing any warp field changes `makeWarpCacheKey` and forces a re-render; re-applying the same edit yields a cache hit (`cacheHit == true`).
- **RT-safety:** the warp swap/playback path passes the P06 `RealtimeCallbackAudit` (`Performance.hpp:55`) with zero allocations/locks on the callback, and the existing TSan CI job stays clean.

## Notes For Agents
- **Realtime law:** no allocation/lock/FFT/file-IO on the audio callback. The stretcher and resampler run off-thread; results land in preallocated grain/ring buffers and reach the callback only through the P05 validated graph snapshot. `renderGraphBlock` (`AudioEngine.cpp:387`) already enforces bounded node/connection counts and uses preallocated `realtimeGraphBuffers_` (`AudioEngine.hpp:140`); keep the warp path inside that discipline — do not add a per-block stretch loop into the `GraphNodeKind::Sample` branch at `AudioEngine.cpp:501-510`.
- **Key bug to fix:** `Warp.cpp:372-384` is a single-read-head resampler; `interpolatedSample` (`Warp.cpp:41-60`) only changes interpolation order by `StretchQuality`. There is currently *no* overlap-add / grain / phase-vocoder anywhere — `grep` confirms it. Build the actual stretch kernel; do not just improve interpolation.
- **Live path has zero warp today:** `sampleNodeValue` (`AudioGraph.cpp:67-92`) and `engineSampleNodeValue` (`AudioEngine.cpp:87`) read `node.samples[sourceFrame]` 1:1 with `sourceFrame = sourceOffsetSamples + relativeSample`. They have no rate field and no warp awareness. The clean approach is to make the warped buffer the node's `samples` (precomputed) so these readers stay 1:1 and RT-safe.
- **GraphCompiler integration:** sample nodes are built in `makeClipNode` (`GraphCompiler.cpp:175-211`); `node.sampleChannels/sampleFrames/samples` come straight from `readPcm16Wav` (`GraphCompiler.cpp:206-210`) with no SR field — add asset SR plumbing there and pass `WarpState` (held in `WarpStateStore`, `Command.hpp:877`) into the compile step.
- **Cache plumbing already exists, unused in render:** `makeWarpCacheKey` (`Warp.cpp:296`), `findValidRenderCacheEntry` (`Warp.cpp:311`), `makeWarpRenderPlan` (`Warp.cpp:321`), `warpRenderPlansAgree` (`Warp.cpp:394`), and `upsert/invalidateRenderCache` (declared `Warp.hpp:115-116`). Wire them; don't reinvent.
- **Existing tests to extend, not duplicate:** `bootstrap_tests.cpp:5358-5410` already exercises plan/cache/agreement at the *planning* layer; add the spectral/DSP-correctness layer on top.
