# P09 Faithful GraphCompiler: render real user audio/MIDI content and add an instrument/synth graph node kind

> **Area:** audio-engine · **Severity:** blocker · **Effort:** L · **Phase:** Phase 2 — Faithful Engine & Content Rendering
> **Depends on:** P05, P07, P08 · **Blocks:** P10, P12, P15, P21, P24, P28, P33

## Objective
Make offline render, stem export, bounce, inspect, and verify reflect the user's *actual* project instead of a fixture. Today `compileProjectAudioGraph` turns every non-asset clip into an FNV-hash sine (`clipFixtureFrequency`) and only synthesizes MIDI when `dataId == "starter-bass-midi"`; there is no instrument graph node. After P09 the compiler resolves arbitrary MIDI clips generically from P08 sidecars, plays real decoded audio-asset samples honoring `startSample`/`sourceOffset`/`length`/fades/`gain`/`reverse`, renders arbitrary MIDI through a new instrument/synth `GraphNodeKind`, and removes the synthetic test-tone/starter-bass special cases from the production path (gated behind an explicit fixture flag). All asset read paths share a single path-traversal guard. This is the precondition for every timeline/audio/MIDI UI and render tool that must show real data.

## Dependencies
- `P05 — Validated audio-graph snapshot, precompiled noexcept render path, and true lock-free SPSC command queue`
- `P07 — Generic command-authored multitrack session API (add/remove/rename tracks, audio+MIDI clips, sends, automation, markers) with dirty-tracking and Save-As`
- `P08 — Versioned binary sidecar format for MIDI, automation, plugin state, and waveform/analysis caches, with migration registry and autosave/recovery`

## Decisions To Make
- **MIDI data delivery into the session lib.** `MidiClipStore` lives in `libs/commands` and depends on `libs/session`; the compiler is in `libs/session`. Inverting that dependency would create a cycle. Settle on a resolver callback / pre-loaded `std::vector<MidiClipData>` injected via `GraphCompileOptions`, populated from P08's `parseMidiClipData` by the caller (Export/CLI/document layer).
- **Instrument voice quality.** A bare sine per note is fixture-grade and aliases at high pitch. Decide the production default: band-limited/poly-BLEP oscillator, a small built-in sampler instrument, or deferring to a real plugin instrument (P12). P09 should ship at least an anti-aliased default voice with per-note velocity→gain and a short release to avoid clicks; richer instruments are P12/P16.
- **Sample-rate mismatch policy.** `readPcm16Wav` returns the asset's own `sampleRate` (`WavFile.hpp:10-14`) but the graph renders at `EngineConfig.sampleRate`. Decide whether P09 errors on mismatch, plays at the wrong pitch (current implicit behavior), or stubs a hook for P29 resampling. Recommend: explicit error/warn now, real resampling in P29.
- **Asset format scope.** Only `.wav` import/read exists (`Assets.cpp:182-184`, `readPcm16Wav`). Decide whether P09 stays WAV-only and defers AIFF/CAF/FLAC/MP3 decode (and AUv3 vs VST3 instrument priority) to P11/P12, or pulls in JUCE `AudioFormatManager` now. Recommend WAV-only here.
- **Fixture gating semantics.** Decide exactly what `allowFixtureTones` permits and where it is *forbidden* (production render/export/inspect/MCP) so the hash tone can never re-enter a real bounce.

## Compromises And Tradeoffs
- **Accepted:** default instrument voice is a simple synthesized oscillator, not a real instrument plugin; full instrument hosting is P12/P16. Velocity maps linearly to gain; no per-voice filtering/ADSR shaping beyond click avoidance.
- **Accepted:** WAV-only asset playback and no sample-rate conversion in P09 — mismatched-SR assets are flagged, not resampled (resampling is P29). Time-stretch/warp and pitch-shift stay out of scope (P28).
- **Accepted:** comp/take-segment selection is *not* rendered here; P09 renders the whole clip body. P33 layers take selection on top of this faithful path.
- **Rejected MVP shortcuts:** keeping `clipFixtureFrequency` as a fallback "so something is audible"; keeping the `"starter-bass-midi"` string match; opening asset files with the raw `projectRoot / relativePath` join (the current traversal hole at `GraphCompiler.cpp:206` and `Export.cpp:104,127`).
- **Risks later tasks must revisit:** automation is not yet applied to instrument/sample nodes (P18); plugin processing on clip→track is absent (P12); bus/send metering and channel-strip DSP are still gain-only (P15); streaming/large-file reads load whole assets into RAM via `readPcm16Wav` — fine offline, must become streaming for P10/P17.

## Acceptance Gates
- `cmake --build --preset unix-debug && ctest --preset unix-debug` passes, including the new render-fidelity cases.
- **Distinct MIDI determinism:** two MIDI clips with different `dataId`/pitch/velocity/position render two different, byte-stable PCM buffers across repeated runs; transposing one by +12 semitones doubles its rendered fundamental.
- **From-scratch audibility:** a project assembled purely via P07 commands (one MIDI clip + one audio clip, no starter project) renders with `peakAbsoluteSample > 0` and matches a checked-in golden buffer.
- **Audio-clip correctness:** an asset clip with non-zero `sourceOffsetSamples`, `fadeInSamples`, `fadeOutSamples`, `gainDb`, and `reversed=true` reproduces the expected interleaved samples within float tolerance (envelope at fade boundaries, reversed source frame order per `AudioGraph.cpp:74-91`).
- **No hash-tone fallback:** grep proves `clipFixtureFrequency` and `"starter-bass-midi"` are gone from the production path; rendering a MIDI clip whose `dataId` has no sidecar fails with a clear error (not a silent tone), unless `allowFixtureTones` is explicitly set.
- **Path safety:** `lamusica_cli render-project <proj> out.wav` and `inspect-project <proj>` exit non-zero with a clear message when an asset's `relativePath` is `../../etc/passwd`, an absolute path, or empty — verified that no read of the out-of-tree file occurs (ASan/strace-style check or a sentinel-file test).
- RT/safety: the offline render path remains the only place that touches disk/decode; nothing added here runs on the audio callback. Verify the precompiled noexcept render path from P05 still validates the graph (`validateGraph`, `AudioGraph.cpp:119`) and that `Instrument` nodes pass topological ordering.

## Notes For Agents
- **Realtime/safety:** all sidecar reads, WAV decode, and path resolution happen at compile time (`compileProjectAudioGraph`), never on the audio thread. The audio thread only ever sees a validated `AudioGraph` snapshot (P05). Do not add allocation/locks/file-IO/logging to `renderGraph`/`renderGraphRange`.
- **Where to wire MIDI data:** P08 ships `serializeMidiClipData`/`parseMidiClipData` in `libs/session/.../MidiSidecar.hpp` and writes per-`dataId` sidecars (see `docs/planning/tasks/P08-binary-sidecar-format-autosave.md:16,26`). The caller (Export/CLI/document) loads these and hands resolved `MidiClipData` to `GraphCompileOptions`; the compiler must not itself reach into `libs/commands`.
- **Render branch pointers:** `Sine` synth/note loop to move/copy is `AudioGraph.cpp:226-257`; `Sample` playback math (offset/reverse/fades/gain) is `sampleNodeValue` at `AudioGraph.cpp:67-92` — reuse it as the model for the instrument envelope. `GraphNote` is `AudioGraph.hpp:20-25`.
- **Asset resolution today:** `makeClipNode` joins `options.projectRoot / asset->relativePath` and calls `readPcm16Wav` directly (`GraphCompiler.cpp:206`) with no traversal check — this is the hole P09 closes. The reusable normalization/prefix helpers are `Assets.cpp:42-57` (`normalizedAbsolute`, `pathHasPrefix`).
- **Existing test fixtures:** `bootstrap_tests.cpp:2475-2480` asserts the 8-note `starter-bass` pattern (pitch 36, start 0/90000) — port these to drive real sidecar data, not the code-fabricated `makeFirstTrackStarterBassMidi`. CLI surfaces `starterMidiNotes`/`bassTranspose` (`tools/cli/src/main.cpp:759-760`) that read from the readiness path; keep those meaningful against real data.
- **Maps to existing tasks docs/tasks/026-036:** the synthetic-render falsehood is *not* flagged by any 026-036 task. P09 is the engine-fidelity precondition that makes 028 (arrangement timeline UI), 029 (audio clip editing + waveforms), and 030 (MIDI piano-roll/drums) render real project data instead of placeholder tones, and underpins the 022-style render/bounce MCP tools surfaced by 034. It does not replace a single 026-036 doc; it fills the gap beneath them.
