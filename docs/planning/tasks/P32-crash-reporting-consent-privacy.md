# P32 Crash-reporting consent, privacy policy, and telemetry-opt-in surface

> **Area:** realtime-perf-security · **Severity:** medium · **Effort:** S · **Phase:** Phase 8 — Polish, accessibility, i18n, and policy (post-release follow-ons)
> **Depends on:** P27 · **Blocks:** none

## Objective
Ship the user-facing consent and privacy layer that P27's crash-reporting plumbing deliberately omits: an explicit, default-private opt-in gate for crash-report and usage-telemetry submission, a privacy disclosure (in-app and in docs) that enumerates exactly what is collected and where it goes, payload scrubbing that strips file paths, project names, and PII before any upload, and a documented, user-overridable reporting endpoint suitable for an AGPL self-hosted deployment. The concrete production outcome is that no crash or telemetry data leaves the machine without an affirmative, revocable user choice, and that the existing `ApplicationPreferences::shareDiagnostics` flag (today a bare unwired boolean) becomes the enforced consent authority for both the DAW and `lamusica_mcpd`. This resolves the open "crash reporting and telemetry policy" decision in the decision-compromise register; it is the policy half of P27, not an MVP afterthought.

## Dependencies
- `P27 — Release pipeline: universal signing, notarization/stapling, hardened runtime+entitlements, Info.plist, provenance, crash reporting, SBOM, and docs` — ships the capture/symbolication plumbing (dSYM archival, off-audio-thread async-signal-safe crash handler for the DAW and `lamusica_mcpd`, the upload client, and the symbol-upload endpoint) but explicitly defers consent/opt-in to this task and forbids enabling network upload without it (`docs/planning/tasks/P27-release-signing-notarization.md:33,43,57`).

## Deliverables
- **Consent state model (files to change).** Extend `ApplicationPreferences` (`libs/session/include/lamusica/session/ApplicationSession.hpp:262-271`) so the existing `bool shareDiagnostics{false}` (default-off) is accompanied by a tri-state consent record — `enum class DiagnosticsConsent { Undecided, Declined, Granted }` defaulting to `Undecided` — plus `std::string diagnosticsEndpoint` (empty = compiled-in default) and a `bool telemetryEnabled{false}` distinct from crash reports. Keep `shareDiagnostics` as the crash-report gate for backward compatibility with the smoke harness (`apps/daw/src/main.cpp:653-683`).
- **Consent enforcement in validation (file to change).** Augment `ApplicationSession::validatePreferences` (`libs/session/src/ApplicationSession.cpp:2228-2248`) so that `shareDiagnostics`/`telemetryEnabled` cannot be `true` while consent is `Undecided` or `Declined`, and so a non-empty `diagnosticsEndpoint` must parse as `https://` (or be the documented `LAMUSICA_DIAGNOSTICS_ENDPOINT` override). Mirror the existing pattern that rejects `allowMcpProjectMutation` without `mcpEnabled` (`libs/session/src/ApplicationSession.cpp:2229-2231`).
- **Payload scrubbing module (new file `libs/session/src/DiagnosticsScrubber.cpp` + header `libs/session/include/lamusica/session/DiagnosticsScrubber.hpp`).** A pure, allocation-tolerant (non-RT) function that takes a captured backtrace/minidump-metadata blob and returns a scrubbed payload: replace absolute filesystem paths with `<path>`/basename-only, strip the project name and any `*.Project.lamusica` paths (cf. `apps/daw/src/main_juce.cpp:12-23` `uniqueUntitledProjectPath`), strip usernames/home dirs, and whitelist only stack frames, OS version, app version + commit SHA (from P27's `version.hpp`), and crash signal. Do NOT reuse `escapeJson` from `libs/mcp_bridge/src/Orchestration.cpp:15-25` for the JSON envelope — it escapes only `"`/`\` and leaves control bytes raw (a P01-class defect); emit a correctly-escaped envelope or a structured writer.
- **Consent-gated upload client wiring (file to change, owned jointly with P27).** The upload step P27 builds must consult the scrubbed payload and refuse to transmit unless consent is `Granted`; route every send through the scrubber and the configured endpoint. The DAW entry is `apps/daw/src/main_juce.cpp`; the daemon entry is `apps/mcpd/src/main.cpp:11`.
- **In-app consent + disclosure UI (file to change `apps/daw/src/main_juce.cpp`).** The current GUI is five static `StatusPanel`s with no menus or Preferences window (`apps/daw/src/main_juce.cpp:57-160`). Add a first-run consent dialog (default = Declined if dismissed) and a Privacy tab/disclosure pane that the user-manual already promises ("Privacy tab", "diagnostics sharing", `docs/user-manual.md:38-41`) but which does not exist in `main_juce.cpp`. If P19's interactive shell has not yet landed, gate the UI behind a minimal modal so the consent path is testable headlessly via `ApplicationSession`.
- **Privacy disclosure document (new file `docs/release/privacy-and-diagnostics.md`).** Enumerate collected fields (stack frames, OS version, app version + commit SHA, crash signal — and explicitly NOT project content, audio, MIDI, file paths, or usernames), the default endpoint, retention period, and the exact opt-out steps. Link it from `docs/user-manual.md:35-41` and `README.md`, and add it to the `install(FILES ...)` set in `CMakeLists.txt` alongside the notices P27 adds.
- **Endpoint override docs (file to change `docs/release/macos-distribution.md`).** Document `LAMUSICA_DIAGNOSTICS_ENDPOINT` and the preferences field for self-hosting, matching P27's symbol-upload endpoint decision (`docs/planning/tasks/P27-release-signing-notarization.md:33`).
- **Tests (files to change `tests/unit/bootstrap_tests.cpp`, `tests/CMakeLists.txt`; new `apps/daw/src/main.cpp` smoke flag).** Extend the existing preferences smoke (`tests/unit/bootstrap_tests.cpp:3386-3400`, registered at `tests/CMakeLists.txt:25-26`): assert no upload occurs with `Undecided`/`Declined`; assert a scrubbed payload contains no project name, no absolute path, no home dir; assert `validatePreferences` throws when `shareDiagnostics=true` with consent `Undecided`. Add a `--app-session-diagnostics-consent-smoke` flag to `apps/daw/src/main.cpp` mirroring the existing flag dispatch (`apps/daw/src/main.cpp:718`).

## Decisions To Make
- **Default posture: silent default-off vs first-run prompt.** Both satisfy "no data without consent"; the prompt improves opt-in rates but adds a first-launch modal. The register's recommended default is "explicitly consented diagnostics" (`docs/planning/decision-compromise-register.md:584,589`) — recommend first-run prompt defaulting to Declined-on-dismiss.
- **Crash reports vs usage telemetry as one toggle or two.** Crash reports (rare, high-value, low-PII once scrubbed) differ from continuous usage telemetry (higher PII surface, AGPL-trust sensitivity). Recommend two independent opt-ins; do not bundle telemetry into the crash consent.
- **Endpoint ownership for an AGPL project.** Decide whether the project runs a hosted endpoint, ships no default endpoint (upload disabled unless self-configured), or both. Must align with P27's symbol-upload endpoint (`docs/planning/tasks/P27-release-signing-notarization.md:33`) and the AGPL corresponding-source posture (`LICENSE:1-9`).
- **Retention and identifier policy.** Whether to attach any anonymous install/session id (re-identification risk vs deduplication value), and the retention window to publish in `docs/release/privacy-and-diagnostics.md`. Recommend no persistent device id and a short, stated retention window.
- **Scrubbing strictness vs symbolication fidelity.** Aggressive path stripping can remove symbol-resolution hints; decide whether to keep module/dylib basenames (needed to match archived dSYMs) while stripping user-specific path prefixes.

## Compromises And Tradeoffs
- **Accepted:** consent state lives in `ApplicationPreferences` and is validated session-side, not in a separate OS keychain entry; acceptable because the flag is non-secret and revocable, and it reuses the existing validated preferences path (`libs/session/src/ApplicationSession.cpp:1938-1940`).
- **Accepted:** the in-app disclosure may ship as a static pane sourced from `docs/release/privacy-and-diagnostics.md` rather than a live policy fetch; avoids a network dependency at launch and keeps the disclosure deterministic and offline-readable.
- **Rejected MVP shortcuts:** shipping with `shareDiagnostics` defaulting to anything but `false`; uploading any payload while consent is `Undecided`; reusing the lossy `escapeJson` (`libs/mcp_bridge/src/Orchestration.cpp:15-25`) for crash JSON; a hardcoded non-overridable endpoint; a privacy "policy" that exists only as prose with no enforced scrubber.
- **Risk later tasks must revisit:** scrubbing is heuristic — minidumps may embed paths/strings in raw memory regions that regex/basename stripping misses; a later task should add a fuzz/round-trip test (leaning on P02's fuzz harness style) asserting no known PII tokens survive scrubbing. Telemetry event schema, if telemetry is enabled, is out of scope here and must be designed before any telemetry event is emitted. This covers the gap "Crash-reporting privacy/consent and telemetry policy uncovered: P27 uploads crash data with no consent/disclosure; register leaves the policy open."

## Acceptance Gates
- `cmake --build --preset <default> && ctest` pass, including the extended preferences smoke and the new `--app-session-diagnostics-consent-smoke` flow.
- With consent `Undecided` or `Declined`, an induced crash in the DAW and in `lamusica_mcpd` (using P27's handler) writes a local backtrace/minidump but performs **zero** network upload — verified by intercepting the upload client (no socket open / no request to the endpoint).
- Setting `shareDiagnostics=true` (or `telemetryEnabled=true`) while consent is `Undecided`/`Declined` throws from `ApplicationSession::setPreferences` → `validatePreferences`; granting consent then enabling succeeds.
- A scrubbed payload generated from a project whose path/name contain a username and `LaMusica Untitled.Project.lamusica` (`apps/daw/src/main_juce.cpp:14-17`) contains none of: the home-dir prefix, the username, the project name, or any absolute path; it still contains stack frames, OS version, and app version + commit SHA.
- The privacy disclosure (`docs/release/privacy-and-diagnostics.md`) enumerates every collected field, the endpoint, retention, and opt-out steps; it is reachable in-app (Privacy pane) and installed into the package (assert via the `VerifyPackage.cmake` required-entries list P27 maintains).
- Setting `LAMUSICA_DIAGNOSTICS_ENDPOINT` (or the preferences endpoint field) redirects uploads to the override; an empty/invalid endpoint with consent granted is rejected by validation, not silently sent to a default.

## Notes For Agents
- **Realtime safety:** this task touches only consent state, scrubbing, and upload — all off the audio thread. The scrubber and JSON writer allocate and must never be called from P27's async-signal-safe handler or the P04/P05 audio callback; the handler writes raw bytes via `write(2)` only, and scrubbing/upload happen later in a separate thread/process (`docs/planning/tasks/P27-release-signing-notarization.md:57`).
- **Grounding pointers:** the consent flag already exists — `ApplicationPreferences::shareDiagnostics{false}` (`libs/session/include/lamusica/session/ApplicationSession.hpp:270`) — but is purely descriptive today; the smoke harness only echoes it (`apps/daw/src/main.cpp:661,682-683`). The validation hook to extend is `ApplicationSession::validatePreferences` (`libs/session/src/ApplicationSession.cpp:2228`). The GUI that needs the Privacy pane is the static panel layout in `apps/daw/src/main_juce.cpp:57-160`; the user-manual already advertises a "Privacy" tab and "diagnostics sharing" that do not yet exist (`docs/user-manual.md:38-41`).
- **JSON gotcha:** do not reuse `escapeJson` (`libs/mcp_bridge/src/Orchestration.cpp:15-25`); it only escapes `"`/`\` and leaves control bytes raw, which is a P01-class on-disk/on-wire defect. Use a correct escaper for the crash envelope.
- **Test framework:** current tests are smoke executables plus `tests/unit/bootstrap_tests.cpp`; P02 migrates them to a real framework. Author the new assertions so they survive that migration (prefer pure functions on the scrubber + `ApplicationSession`, not GUI-only paths).
- **Maps to existing tasks docs:** this completes the consent/privacy half of `docs/tasks/036-release-hardening-signing-docs-examples.md` and resolves the open "crash reporting and telemetry policy" decision in `docs/planning/decision-compromise-register.md:580,584,589`. Neither 036 nor P27 addresses consent/disclosure (`relation_to_existing`); P27 ships the plumbing, P32 ships the policy gate that must land before any crash upload is enabled by default.
