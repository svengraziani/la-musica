# P13 Plugin opaque state persistence, editor windows, and graceful plugin error model

> **Area:** plugin-hosting · **Severity:** blocker · **Effort:** L · **Phase:** Phase 4 — Real Plugin Hosting
> **Depends on:** P08, P12 · **Blocks:** P14, P22

## Objective
Make plugin state a first-class, durable, recoverable part of the project: persist and restore the full opaque `getStateInformation`/`setStateInformation` chunk for every insert and instrument slot (alongside bypass, insert-chain order, and instrument-rack assignment), so reloading a project reproduces the exact internal plugin state and renders bit-equivalent audio. Host real `juce::AudioProcessorEditor` windows (with a generic parameter-panel fallback) at persisted geometry, kept in sync with automation and the session parameter list. Replace the pervasive `throw std::runtime_error` parse path and the substring-based `"bypassed":true` detection with a structured parser and a result/error model, so a single malformed plugin record drops with a per-record diagnostic and the rest of the session loads. Per the mission constraint, only the unavoidably-opaque chunk lives in the binary sidecar; all structural plugin metadata stays in the deterministic JSON manifest.

## Dependencies
- `P08 — Versioned binary sidecar format for MIDI, automation, plugin state, and waveform/analysis caches, with migration registry and autosave/recovery` — defines the `PluginState/` sidecar container kind (reserved but stubbed in P08), the CRC32C header, the durable sidecars-before-manifest write order, and the `MigrationRegistry` this task hooks a `v2->v3` step into.
- `P12 — Plugin graph node + RT-safe parameter/bypass delivery in the render path` — provides the live plugin instance held by the graph node whose `getStateInformation`/`setStateInformation` we call, and the RT-safe parameter/bypass delivery queue that editor edits and state restores must feed through (never direct from the message thread to the audio thread).

## Decisions To Make
- **Manifest-vs-sidecar split (mission constraint, from corrections):** confirm the firm rule that *only* the opaque chunk is binary; bypass, insert order, instrument assignment, parameter list, and the state id+checksum stay in the deterministic JSON manifest. Add it as an explicit acceptance criterion shared with P08 so the bundle is never "opaque binary-only."
- **Format identity on the chunk:** AU and VST3 chunks are not interchangeable. Decide whether the sidecar header records the producing plugin's `identifier` + `format` + plugin version so a restore into a mismatched/upgraded plugin is detected and downgraded to a parameter-only restore rather than fed garbage to `setStateInformation`.
- **Editor priority & licensing:** which editor path ships first — AU custom UI (no SDK license issue) vs VST3 (gated today by `vst3SdkAvailable`/`vst3LicenseAccepted` in `PluginHostEnvironment`, `Plugin.hpp:80-86`); and the generic fallback policy when `hasEditor()` is false. Note the GPL/AGPL surface: bundling no third-party plugin binaries here, only hosting user-installed ones.
- **State capture timing:** snapshot the chunk on save / on editor close / on a debounced timer? `getStateInformation` may allocate and must run on the message thread, never the audio thread — pick the trigger and the staleness window.
- **Parameter-vs-state authority on reload:** if both the persisted parameter list and the opaque chunk exist, which wins (chunk is authoritative; parameter list is a degraded fallback and an automation/discovery aid). Define reconciliation when they disagree.
- **Migration backfill:** pre-state-blob projects have inserts with no `stateSidecarId`. Decide the migration: leave it empty (parameter-only restore, no sidecar) and only create a sidecar on next save, vs synthesize an empty chunk.

## Compromises And Tradeoffs
- **Accepted:** the opaque chunk is not human-diffable or hand-editable — unavoidable for AU/VST3. Mitigated by VCS-diffable-by-reference (stable per-insert sidecar id + checksum in JSON) and a `--dump-plugin-state` debug path; this satisfies the mission's "not opaque binary-only session files" rule because the structural truth stays in `project.json`.
- **Accepted:** chunk equality across plugin-version upgrades is not guaranteed; we detect format/version mismatch and fall back to parameter-only restore with a diagnostic rather than blocking the load.
- **Rejected MVP shortcut:** persisting only the flat `parameterValues` list (today's behavior) — it loses synth sample slots, internal modulation, sub-presets, and anything not surfaced as an `AudioProcessorParameter`. Also rejected: keeping the substring bypass hack (`Plugin.cpp:183`) and the lenient numeric parse, both of which are correctness/security bugs flagged in `covers_gaps`.
- **Risk later tasks must revisit:** P14 (freeze/bounce) must capture state at freeze time and survive thaw; P22 (mixer/plugin UI) consumes the editor-window model and per-record diagnostics surface; P29 (sample-rate reconciliation) — a plugin's chunk may encode SR-dependent state, so a SR change after restore may need a re-prepare. Reserve the format/version header fields now.

## Acceptance Gates
- `cmake --build build && ctest --test-dir build --output-on-failure` passes, including `plugin_state_tests`.
- **Full-state round-trip:** save a project with a stateful plugin, mutate its internal state, save again, reload — the restored chunk is byte-equal (and `stateChunkChecksum` matches) and an offline render through the GraphCompiler is sample-equivalent to pre-reload. Two consecutive saves of unchanged state produce byte-identical sidecar files (determinism).
- **Structural metadata stays JSON:** assert that `project.json` after save contains the bypass flag, insert order, instrument assignment, and the state id+checksum as text, and that the only binary artifact is under `PluginState/` (mission-constraint gate, shared with P08).
- **Graceful error model:** a project with one malformed plugin record (bad number, missing key, unterminated string) loads every other track and insert and returns a per-record diagnostic; no exception escapes `parseProjectManifest`/the document-open boundary.
- **Parser fuzz (from acceptance_highlights):** a parameter whose *string value* contains `"bypassed":true` does NOT set `insert.bypassed` (regression for `Plugin.cpp:183`); `"value":1.2.3` and `"x":12abc` are rejected (regression for `:68-90` trailing-garbage); truncated/duplicate-key objects drop the single record, not the chain.
- **Editor windows:** opening an insert shows its real custom UI, or the generic parameter panel when `hasEditor()` is false, at the persisted `x/y/width/height`; a control edit changes rendered audio and updates the session parameter value; closing releases the editor with no leak (run under ASan/leak check) while the processor keeps playing.
- **Migration:** a pre-state-blob fixture (no `stateSidecarId`) migrates forward, loads parameter-only with a diagnostic, and gains a sidecar on next save; a future-version sidecar is rejected via P08's `MigrationRegistry`.

## Notes For Agents
- **Realtime safety:** `getStateInformation`/`setStateInformation`, editor creation, sidecar I/O, JSON, CRC32C, and `MemoryBlock` allocation are all message-thread/compile-time work — NONE may run on the audio callback. Restored state reaches the audio thread only through P12's validated snapshot + RT-safe queue; the audio thread touches only preloaded, owned bytes. Editor parameter edits go message-thread → P12 parameter queue → render path, never a direct write to a live processor from the UI.
- **Current reality (verified):** `PluginInsert` (`libs/session/include/lamusica/session/Plugin.hpp:99-104`) persists only `parameterValues` — there is no opaque-state field and no plugin bytes are written to any bundle today. The substring bypass bug is `libs/session/src/Plugin.cpp:183`; the lenient numeric parse is `:68-90` (`from_chars` ignores the residue); the parse path throws `std::runtime_error` pervasively (`:28,:32,:39,:61,:75,:87,:100,:106,:146`). The insert chain lives in `PluginInsertChainStore` (`libs/commands/include/lamusica/commands/Command.hpp:785-792`), separate from the manifest's bare `PluginReference{id,trackId,format,identifier}` (`ProjectManifest.hpp:99-104`) — wire the state id through both.
- **Reuse the existing result type:** `CommandResult{bool ok; std::string message}` (`libs/commands/include/lamusica/commands/Command.hpp:19-21`) is the established result/error precedent — model `PluginParseResult` on it rather than inventing a divergent shape. Reuse `readRequiredBool` (`Plugin.cpp:92-101`) for the bypass fix; it already exists and is correct.
- **Sidecar plumbing:** P08 reserves the `PluginState/` container kind and the `PluginState/` bundle subdir; P13 is its first real producer. Use P08's `writeSidecar`/`readSidecar` and its sidecars-before-manifest durable order so a crash between writes is detected by checksum, not silently used. Mark plugin-state sidecars NOT regenerable (unlike the `Analysis/` caches).
- **Editor geometry already modeled:** `PluginEditorWindow` (`Plugin.hpp:127-136`) and `serialize/parsePluginEditorState` (`Plugin.cpp:654-689`) already persist `open/pinned/x/y/width/height` and validate positive dimensions (`openPluginEditor`, `:628-643`); P13 binds these to live `juce::AudioProcessorEditor` windows in `apps/daw/src/main_juce.cpp`, it does not redesign the model.
- **Maps to existing docs:** fulfills `docs/tasks/031-plugin-hosting-ui.md` acceptance "Plugin state saves, reloads, and participates in undoable insert/remove/reorder commands" (`:23`), "Parameter IDs remain stable enough for automation across reloads" (`:24`), and the plugin-window deliverable (`:16`). The opaque-chunk container itself is owned by P08 (`docs/planning/tasks/P08-binary-sidecar-format-autosave.md:34`); the live plugin instance + RT delivery by P12. P13 supersedes 031's hand-wave that the existing `serializePluginInsertChain` scaffolding is "real" plugin-state persistence.
