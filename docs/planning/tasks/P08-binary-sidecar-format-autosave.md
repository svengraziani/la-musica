# P08 Versioned binary sidecar format for MIDI, automation, plugin state, and waveform/analysis caches, with migration registry and autosave/recovery

> **Area:** session-model · **Severity:** blocker · **Effort:** L · **Phase:** Phase 2 — Durable Production Session Model
> **Depends on:** P01, P07 · **Blocks:** P09, P13, P23, P29

## Objective
Make the project bundle a real, durable, versioned container: move large/binary data (MIDI note/CC/pitchbend/aftertouch/program data, automation point sets, plugin opaque state, and waveform/analysis caches) out of synthesized-in-code stubs and inline JSON into deterministic, checksummed binary sidecars under the bundle, referenced by stable id from `project.json`. Replace the single hardcoded `v0->v1` upgrade with a chained migration registry that rejects unknown future versions, and add periodic autosave to a recovery artifact plus startup crash-recovery detection. The outcome is a production format where a crash between manifest and sidecar writes is *detected* (never silently used), large MIDI clips persist losslessly, and the engine reads real user content from disk rather than `makeFirstTrackStarterBassMidi()`.

## Dependencies
- `P01 — Fix on-disk format integrity: schema conformance, lossless/locale-independent floats, JSON escaping, durable atomic save` — provides the trustworthy serializer, RFC8259 escaping, and the `fsync`/`F_FULLFSYNC`+dir-`fsync` durable-save primitive that the sidecar write flow extends.
- `P07 — Generic command-authored multitrack session API (add/remove/rename tracks, audio+MIDI clips, sends, automation, markers) with dirty-tracking and Save-As` — provides the dirty/clean state machine that autosave keys off, the command-authored mutation surface that produces real MIDI/automation content to persist, and `saveProjectAs(path)` whose bundle copy must include sidecars.

## Decisions To Make
- **Endianness & checksum:** standardize little-endian payloads (native to Apple Silicon/x86_64; no byte-swap on macOS) with CRC32C (hardware-accelerated) vs CRC32 — pick one and freeze it in the header magic.
- **Sidecar granularity:** one sidecar file per MIDI clip and per automation lane (good for diffs, Save-As copy, partial autosave) vs one packed container (fewer files, fewer `fsync`s). Recommend per-id files keyed by stable `dataId`.
- **Schema-version semantics:** does adding sidecars bump the *project* schema to v2, or is the sidecar container independently versioned? Decide whether `MidiClipReference.transposeSemitones` and other v1-illegal fields (flagged by P01 against `additionalProperties:false`) are folded into the v2 schema now.
- **Plugin state & analysis caches scope:** define and version the `PluginState/` and `Analysis/` container kinds here even though producers land later (plugin opaque blobs in P13, waveform/peak caches consumed by P21/P23). Caches must be marked *regenerable* so corruption triggers rebuild, not project failure.
- **Autosave cadence & artifact location:** interval/idle-trigger policy, and whether the recovery artifact lives inside the bundle (`.recovery/`) or in an app-support sandbox path (matters for P27 sandbox confinement and P25 daemon).
- **Migration irreversibility:** migrations are forward-only; decide whether to write a pre-migration backup copy of the bundle before in-place upgrade.

## Compromises And Tradeoffs
- **Accepted:** binary sidecars are not human-diffable; mitigated by stable per-id files, a documented layout, and a `--dump-sidecar` debug path. The manifest stays JSON for source-control friendliness (per 005's "stable for diffs" goal).
- **Accepted:** plugin-state and analysis container *kinds* are defined now but only stubbed/unit-tested; real producers arrive in P13 (plugin opaque state) and the cache consumers in P21/P23. We define the format once to avoid a second migration.
- **Rejected MVP shortcut:** continuing to synthesize note data in `StarterProject.cpp` / hardwire `"starter-bass-midi"` in `GraphCompiler.cpp:74`. P09 cannot render real user MIDI until sidecars are the source of truth, so this debt is paid here.
- **Rejected:** inlining MIDI/automation as JSON arrays in `project.json` (10k notes would bloat the manifest, lose float fidelity, and break diff stability).
- **Risk later tasks must revisit:** P29 (sample-rate reconciliation) may require sidecars to record their authored sample rate so MIDI/automation `samplePosition` values can be rescaled on SR change — reserve a header/manifest field. P14 freeze/bounce will add a large cache kind; ensure the container's `uint64` length and alignment headroom suffice.

## Acceptance Gates
- `cmake --build build && ctest --test-dir build --output-on-failure` passes, including the new sidecar/migration/recovery suites.
- **MIDI round-trip exactness:** a clip with notes, CC, pitchbend, aftertouch, and program changes serializes and re-parses byte-for-byte equal (deep `MidiClipData` equality); `value`/float fields survive via raw bit pattern.
- **Large-clip budget:** a 10,000-note clip round-trips under an asserted byte-size and wall-time budget; the test fails if either is exceeded.
- **Corruption detection:** flipping one payload byte, truncating the file, zeroing the length field, or deleting a referenced sidecar (orphan) each causes `open()` to throw an actionable error — never a silent partial/default load. Add a fuzz-style truncation sweep.
- **Migration matrix:** a `v0` fixture migrates `v0->v1->v2` with zero data loss (assert structural equality to the expected v2 fixture); a `v2`-and-future fixture with `schemaVersion=99` is rejected with "Unsupported project schema version"; a non-contiguous gap is rejected.
- **Crash recovery:** simulate a crash by writing the recovery artifact then aborting before clean save; on reopen, `hasPendingRecovery()` is true and unsaved edits are recoverable. A subsequent clean `save()` calls `clearRecovery()` and `hasPendingRecovery()` becomes false.
- **Durability ordering:** a fault-injection test confirms sidecars are durably written (`F_FULLFSYNC`) and renamed *before* `project.json`, so a manifest referencing a not-yet-flushed sidecar can never occur.
- **Determinism:** identical in-memory `MidiClipData`/automation produces byte-identical sidecars across two runs and two locales (extends P01's locale test to binary payloads).
