# P14 Engine-wide plugin delay compensation (PDC) and track freeze/bounce-to-cache

> **Area:** audio-engine · **Severity:** high · **Effort:** L · **Phase:** Phase 4 — Plugin Hosting & Engine DSP Completion
> **Depends on:** P12, P13 · **Blocks:** P17, P26

## Objective
Make the mix sample-accurate and CPU-economical at production scale: query every node's reported processing latency (plugins included), propagate it through the compiled graph, and insert compensating delay so all paths converging on a summing/output node are sample-aligned, while reporting total engine output latency to transport and recording. Add non-destructive track freeze that renders a track's source + insert chain to a cached audio file and substitutes cached playback for live processing (reversible, bit-similar on unfreeze, with frozen state persisted and the cache invalidated when source or inserts change), plus a bounce-to-audio path that honors PDC and automation. This is the full DAW behavior implied by the mission's per-track "freeze, bounce" mandate (`docs/mission/product-mission.md:13`), not a stub.

## Dependencies
- `P12 — Plugin graph node + RT-safe parameter/bypass delivery in the render path` — supplies the plugin graph node kind whose `getLatencySamples()` PDC must query and whose live processing freeze substitutes. PDC is meaningless until plugins are actually audible in `renderGraphBlock`.
- `P13 — Plugin opaque state persistence, editor windows, and graceful plugin error model` — supplies the persisted plugin state and the error model; freeze must capture a deterministic render of a fully-restored insert chain, and a plugin in error state must not silently corrupt a freeze render.

## Decisions To Make
- **PDC scope & rounding:** compensate only paths into shared summing nodes, or full graph-wide alignment to `max(latency)` at the output? Plugin latency can be fractional at the host SR; decide round-to-nearest-sample vs sub-sample fractional-delay (reject fractional for v1, document the residual error budget).
- **Freeze cache format & channel count:** freeze to PCM24 WAV (matches `ExportBitDepth::Pcm24` and the project sample rate from P29) vs PCM32-float for exact unfreeze reproduction. Bit-similar (within dither/round-trip epsilon) vs bit-identical — float caches make "identical on unfreeze" achievable; PCM24 trades exactness for size.
- **Freeze granularity:** freeze the post-insert pre-fader signal (so mixer fader/pan stay live and automatable) vs freeze the full strip. Post-insert pre-fader is the production-correct choice and must be stated.
- **Cache invalidation fingerprint inputs:** which fields invalidate a freeze (clip edits, asset content hash, insert add/remove/param change, plugin state blob from P13, project SR)? Plugin parameter automation under the frozen region must invalidate or be baked — decide and test.
- **Freeze cache location & lifecycle:** inside the bundle (`Analysis/`-style sidecar, copied by Save-As, in source control's `.gitignore`) vs app-support scratch (smaller bundle, lost on machine move). Tie to P08's regenerable-cache kind.
- **Engine latency reporting contract:** is `engineLatencySamples()` the value P17 uses verbatim for record alignment, or does recording add its own input latency on top (it does — `RecordingOptions.measuredInputLatencySamples`, `libs/audio/include/lamusica/audio/Recording.hpp:19`)? Define the sum precisely so P17 does not double-count.

## Compromises And Tradeoffs
- **Accepted:** v1 PDC is integer-sample only; sub-sample plugin latency leaves a bounded phase residual. Documented and revisited only if a fractional-delay requirement surfaces.
- **Accepted:** freeze renders the whole track timeline span (not just the audible region), trading disk/time for simple, deterministic substitution and trivially-correct unfreeze.
- **Accepted:** freeze cache kind reuses P08's regenerable sidecar container; corruption triggers a rebuild (re-render), never a project-load failure.
- **Rejected MVP shortcut:** "freeze" that merely bypasses processing or zeroes latency. That defeats the CPU-reduction and sample-alignment goals (`covers_gaps`: "No track freeze / bounce-to-cache substitution path in the engine"). The substitution must be a real cached `Sample` node in the compiled graph.
- **Rejected:** computing/allocating compensation on the audio callback. All solving and ring-buffer sizing is off-thread; the RT path only indexes preallocated storage (engine RT rules; `renderGraphBlock` already throws on unbounded graphs at `libs/audio/src/AudioEngine.cpp:391`).
- **Risk later tasks must revisit:** P17 (latency-compensated recording) depends on a correct `engineLatencySamples()`; P29 (project SR) means a freeze rendered at one SR is invalid at another — `FrozenTrackState.frozenAtSampleRate` exists to force re-freeze. P26's deadline suite must include a worst-case (max plugins + max compensation) block-time gate. P28 (warp/stretch) bounce must also flow through this PDC-aware bounce path.

## Acceptance Gates
- `cmake --build build && ctest --test-dir build --output-on-failure` passes, including the three new suites.
- **Phase alignment (null test):** a graph with two parallel paths to one summing node, one path carrying a fixed-latency mock node, sums to a comb-filtered (partially-cancelling) result with PDC off; with PDC on, the two paths null to within an asserted epsilon (no comb filtering). The test asserts the spectral/peak difference both ways.
- **Glitch-free recompute:** enabling/disabling a high-latency mock plugin recomputes compensation off-thread and the new snapshot is applied atomically; a continuity test asserts no sample discontinuity (no zero-fill, no doubled samples) across the swap block, and other tracks' output is byte-identical across the swap.
- **Engine latency reported:** `engineLatencySamples()` equals the longest-path latency to the output node for a known fixture; bounce output is head-trimmed so its first audible sample lands at sample 0 (latency-free file).
- **Freeze CPU reduction:** a frozen track's compiled subgraph contains exactly one `GraphNodeKind::Sample` cache node (no clip/insert nodes); a render-time measurement (reuse the harness in `libs/session/src/Performance.cpp:266`) shows lower callback time than the live chain.
- **Unfreeze fidelity:** unfreeze restores the live subgraph and re-renders bit-similar (within the chosen cache epsilon) to the pre-freeze output of the same span.
- **Cache invalidation:** mutating a clip on the track, an asset's content, or an insert parameter/order flips `isFreezeCacheValid(...)` to false; an unrelated edit on another track keeps it valid.
- **Persistence:** `frozenTracks` + cache survive save/load (round-trip through `serializeProjectManifest`/`parseProjectManifest` and the P08 sidecar copy); reopening a frozen project plays from cache without re-rendering.
- **Determinism/RT:** two freeze renders of the same chain produce byte-identical caches; the PDC solver allocates nothing on the RT path (verify under P06's harness/TSan job once available).

## Notes For Agents
- **Realtime constraints:** all PDC solving, ring-buffer sizing, freeze rendering, fingerprinting, and disk I/O happen at compile/snapshot time or on a worker thread — never in `renderGraphBlock`. The render path may only read preallocated buffers; `renderGraphBlock` already enforces bounded node/connection counts and throws otherwise (`libs/audio/src/AudioEngine.cpp:391-399`). Size compensation ring buffers from the solver and fold them into the `realtimeGraphBuffers_` preallocation (`libs/audio/include/lamusica/audio/AudioEngine.hpp:140`); do not `new`/`resize` in the callback.
- **Where latency does not exist yet:** `GraphNode` (`libs/audio/include/lamusica/audio/AudioGraph.hpp:27`) and `GraphConnection` (`:46`) carry no latency fields today, and `GraphNodeKind` (`:12`) has no plugin kind — P12 adds the plugin node and `getLatencySamples()` source; this task adds the fields and the propagation. Do not invent a latency query that P12 has not landed; gate on it.
- **Reuse the existing scheduler:** `renderGraphBlock` already computes a Kahn topological order (`libs/audio/src/AudioEngine.cpp:427-453`) and sums sources into destinations at `:512-532`. The PDC solver should mirror that traversal off-thread to compute longest-path cumulative latency; the only render-path change is inserting the per-connection delay ring before the accumulate at `:529`.
- **Compile-time integration point:** populate compensation in `compileProjectAudioGraph` just before `validateGraph` (`libs/session/src/GraphCompiler.cpp:324`); the freeze substitution mirrors `makeClipNode` (`:175-212`) which already builds a `GraphNodeKind::Sample` node from a WAV (`audio::readPcm16Wav` at `:206`) — the freeze cache node is the same shape, sourced from the cache file.
- **Bounce currently ignores PDC:** `renderGraphRange` (`libs/audio/src/Bounce.cpp:113`) calls `renderGraph` with the raw start/frames and no latency handling. Extend the span and trim the head by `totalEngineLatencySamples` so bounced files are aligned.
- **Recording handshake:** `RecordingOptions.measuredInputLatencySamples` (`libs/audio/include/lamusica/audio/Recording.hpp:19`) and `latencyAlignedStart` (`libs/audio/src/Recording.cpp:35`) already align input latency; this task supplies the *output/processing* latency that P17 combines with it. Keep the two sources distinct to avoid double-compensation.
- **Persistence reuse:** add `frozenTracks` to `ProjectManifest` (`libs/session/include/lamusica/session/ProjectManifest.hpp:122`) alongside `plugins`/`automation`; the freeze cache is a P08 regenerable sidecar kind (P08 explicitly reserves the "large cache kind" for P14 — `docs/planning/tasks/P08-binary-sidecar-format-autosave.md`). Serialize via the existing manifest JSON path; do not inline the audio.
- **Maps to existing docs:** delivers the per-track "freeze, bounce" called out in `docs/mission/product-mission.md:13` and the freeze/bounce implied by `docs/tasks/015-mixer-routing.md` / `docs/tasks/032-mixer-routing-automation-ui.md` (mixer). No `docs/tasks/026-036` task names PDC explicitly — P14 is its sole owner — and it extends the plugin-hosting scope of `docs/tasks/014-plugin-hosting.md` with the latency/freeze behavior that 014 does not cover. The bounce path supersedes the latency-naive `bounceGraphToWav` from `docs/tasks/022-mcp-audio-render-tools.md`'s render surface.
