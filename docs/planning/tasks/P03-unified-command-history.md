# P03 Unify all edit-domain commands under one polymorphic ICommand history with serialization, coalescing, bounded depth, and graceful error handling

> **Area:** commands · **Severity:** blocker · **Effort:** XL · **Phase:** Foundation (deterministic core)
> **Depends on:** P01 · **Blocks:** P07, P24

## Objective
Make the command layer the single deterministic source of truth for every session mutation. All ~40 edit-domain commands (project/clip, MIDI, pattern, automation, mixer, plugin, warp) implement one polymorphic `ICommand` interface and live in one ordered `CommandHistory`, replacing the four parallel `*EditHistory` stacks in `EditTools.cpp`. Every command gains a true `redo()` that restores captured post-apply state (no re-rolling humanize RNG, no re-thinning automation), a per-command schema version with a migration hook, gesture-keyed coalescing, a bounded undo depth/memory budget, and exception-free degradation on malformed/unknown serialized input. This is production-grade undo, not an MVP stub.

## Dependencies
- `P01 — Fix on-disk format integrity: schema conformance, lossless/locale-independent floats, JSON escaping, durable atomic save`. P03 reuses P01's locale-independent float formatting and JSON string escaping for `serialize()`; today none of the `serialize()` bodies in `libs/commands/src/Command.cpp` escape string fields (e.g. `ApplyWarpGrooveCommand::serialize`, `Command.cpp:3501`), so a clip id containing `"` or `\` breaks the substring readers. Serialization byte-stability and round-trip gates in P03 depend on P01's canonical float/escape rules being in place.

## Deliverables
- **Unified interface.** Extend `ICommand` (`libs/commands/include/lamusica/commands/Command.hpp:30`) so every family derives from it: `metadata/validate/preview/apply/undo/redo/serialize` plus new `schemaVersion()`, `kind()`, `mergeKey()` (kind+target+gesture token), `canMerge(const ICommand&)`, `mergeWith(ICommand&&)`. Operate on one mutation surface — see "Decisions" for the store-unification approach.
- **Promote non-ICommand families.** The MIDI (`AddMidiNoteCommand` … `AddMidiProgramChangeCommand`, `Command.hpp:366-624`), pattern (`AddPatternClipCommand`/`DuplicatePatternVariationCommand`/`EditPatternStepCommand`, `Command.hpp:637-696`), automation (`AddAutomationPointCommand`/`CaptureAutomationWriteCommand`, `Command.hpp:708-756`), mixer (`SetChannelMixCommand`, `Command.hpp:765`), plugin (`AddPluginInsertCommand` … `ApplyPluginPresetCommand`, `Command.hpp:795-875`) and warp (`AddWarpMarkerCommand` … `ApplyWarpGrooveCommand`, `Command.hpp:887-990`) classes are currently free `final` classes (no `: public ICommand`) operating on side stores (`MidiClipStore`, `PatternClipStore`, `AutomationLaneStore`, `PluginInsertChainStore`, `WarpStateStore`). Make them `ICommand` subclasses with real `redo()`.
- **`redo()` with post-apply restore.** Replace the default `redo()` that just calls `apply()` (`Command.hpp:40-42`). Each command captures the *post-apply* state at first `apply()` and `redo()` restores it directly. Critical for non-idempotent transforms: `HumanizeMidiClipCommand` (RNG seed at `Command.cpp:1638`), `DuplicatePatternVariationCommand` (`seedOffset_`, `Command.cpp:2403`), and `CaptureAutomationWriteCommand` thinning (`thinningTolerance_`). Today these store only `previousClip_`/`previousLane_` for undo (e.g. `SetMidiNoteLengthsCommand::undo`, `Command.cpp:1680`) and have no captured forward state at all.
- **Full serialization registry.** Extend `registeredProjectCommandNames()` (`Command.cpp:3652`, currently 18 names) and `commandFromSerialized()` (`Command.cpp:3673`) to cover all ~40 types. Add a `"v":<n>` schema-version field to every `serialize()` body and a migration entry point that upgrades older payloads before dispatch.
- **Coalescing + bounds in `CommandHistory`.** Extend `CommandHistory` (`Command.hpp:1010`, impl `Command.cpp:3589-3650`): on `execute`, if the incoming command `canMerge` with `undoStack_.back()` (same `mergeKey`, same open gesture), merge in place instead of pushing. Add `beginGesture(token)`/`endGesture()`, `setMaxDepth`/`setMemoryBudget`, and evict the oldest entries (drop from the front of `undoStack_`) when either bound is exceeded.
- **Exception-free replay/validate.** `commandFromSerialized` currently `throw`s on missing fields (`Command.cpp:68`, `:791`) and `TransactionCommand::validate` (`Command.cpp:3521`) and `replaySerializedCommands` (`Command.cpp:3794`) call it on the hot path. Refactor to return `std::expected<CommandPtr, CommandResult>` (or out-param + `CommandResult`) so malformed/unknown/truncated input yields `ok==false` and never throws.
- **Scope-aware JSON reader.** Replace `readSerializedString/Int64/Float/Bool` (`Command.cpp:64-151`), which all do a first-substring `json.find(pattern)` and so read a nested field with the same key, with a brace/bracket-tracking object scanner (the array scanner at `Command.cpp:153` already tracks depth and is the model). Reuse P01's parser if one is introduced there.
- **Cross-domain `TransactionCommand` + split-clip fix.** `TransactionCommand` (`Command.hpp:992`) must compose commands from any domain. Fix `makeSplitClipCommand` (`Command.cpp:3866`): when the source clip is missing it returns a silent empty transaction no-op (`Command.cpp:3870-3873`) — return a command whose `validate()`/`apply()` reports `ok==false` instead.
- **Delete parallel stacks; rewire `EditTools`.** Remove `MidiEditHistory`/`AutomationEditHistory`/`MixerEditHistory`/`PluginEditHistory` (`libs/mcp_bridge/include/lamusica/mcp_bridge/EditTools.hpp:44-99`, impls `EditTools.cpp:137-330`) and the `std::variant` command aliases (`EditTools.hpp:29-42`); route every edit tool through the single `CommandHistory`.
- **Tests.** New `tests/unit/command_history_tests.cpp` (registered in `tests/CMakeLists.txt:1` alongside `bootstrap_tests.cpp`) under the P02 framework: per-type round-trip, cross-domain undo/redo ordering, coalescing depth, fuzz/corrupt-input degradation.

## Decisions To Make
- **Store unification model.** Either (a) fold `MidiClipStore`/`PatternClipStore`/`AutomationLaneStore`/`PluginInsertChainStore`/`WarpStateStore` into `session::ProjectManifest` so all `ICommand`s share `apply(ProjectManifest&)`, or (b) keep an `EditContext` aggregate passed by reference. (a) is the production target (P07 builds the multitrack session API on a single manifest) and removes the variant/dual-signature split — choose it unless P01/P07 sequencing forbids touching the manifest now.
- **redo() state-capture mechanism.** Snapshot the whole affected sub-object (clip/lane/chain) post-apply vs. store a structured forward-delta. Snapshot is simpler and already half-present (`previousClip_`); deltas save memory and feed the memory budget. Decide per family or pick one rule.
- **Coalescing gesture token.** Who owns the token — caller-supplied gesture id (UI drag, MCP batch) vs. time-window heuristic. Caller-supplied is deterministic and testable (200 MoveClip in one drag → depth 1); reject the time-window shortcut for the core.
- **Memory budget accounting.** Bytes vs. entry count vs. both; how `CaptureAutomationWriteCommand` (unbounded `samples_` vector) is charged. Pick a `sizeBytes()` hook on `ICommand`.
- **Version/migration policy.** Single global `"v"` vs. per-command version; whether unknown future versions hard-fail or best-effort. Tie the field name and bump rules to P01's `project-v1.schema.json` conventions (`docs/schemas/project-v1.schema.json`).
- **Serialized-failure contract for MCP.** Define the exact `CommandResult.message` shape that `replaySerializedCommands`/edit tools surface for malformed input, since P24's JSON-RPC tools depend on it.

## Compromises And Tradeoffs
- **Accepted:** keeping the line-oriented JSON-ish `serialize()` text format (not switching to a real document model) for this task, provided the reader is scope-aware and escaping is P01-correct. A binary command-log format is deferred to P08's sidecar work.
- **Accepted:** snapshot-based `redo()` may hold larger undo entries than deltas; the memory budget bounds the blast radius.
- **Rejected MVP shortcut:** leaving the four `*EditHistory` stacks and only adding `redo()` to ICommand families. That perpetuates non-chronological cross-domain undo and is the exact gap P07/P24 forbid.
- **Rejected:** the silent empty-transaction no-op as "graceful" behavior — it hides errors from MCP callers and must become an explicit failure.
- **Risks later tasks must revisit:** this `CommandHistory` is single-threaded and not yet RT-safe; **P05** must keep all command authoring off the audio thread and mirror only validated snapshots into the render path — undo/coalescing logic must never run on the audio callback. **P08** will likely replace the text serialize format with a versioned binary log; the migration hook added here is the seam. **P25**'s addressable undo registry needs stable `commandId`/`auditId` identity preserved across coalescing (a merged entry must expose a deterministic id).

## Acceptance Gates
- `cmake --build build --target lamusica_unit_tests && ctest --test-dir build -R command_history` passes, including:
  - A 6-command cross-domain sequence (e.g. add_track → add_clip → add_midi_note → set_channel_mix → add_plugin_insert → add_automation_point) undone six times then redone six times yields bit-identical manifest state at every step, in strict chronological order across domains.
  - Every command type round-trips `serialize() → commandFromSerialized() → serialize()` byte-identically, and replays to identical state. Includes string fields containing `"`, `\`, and non-ASCII (escaping correctness vs. P01).
  - 200 `MoveClip` issued inside one gesture token collapse to `undoDepth()==1`; the same 200 split across two gestures yield `undoDepth()==2`; a single undo of the merged entry restores the pre-gesture position.
  - Truncated, unknown-name, duplicate-key, and special-char serialized inputs each return `CommandResult.ok==false` with a message and **never throw** — assert via a fuzz loop (feeds P02's corrupt-project harness).
  - `makeSplitClipCommand` on a missing clip id produces a command whose `validate()` returns `ok==false` (no empty no-op transaction).
  - Bounded depth: with `setMaxDepth(N)`, executing `N+10` distinct gestures leaves `undoDepth()==N` and the oldest entries evicted; memory-budget eviction verified with a large `CaptureAutomationWriteCommand`.
  - `HumanizeMidiClipCommand` apply→undo→redo reproduces the *first* humanized result exactly (no RNG re-roll); same for `DuplicatePatternVariationCommand` seed and automation thinning.
- `clang-tidy`/build clean; no new `throw` reachable from `replaySerializedCommands` or `TransactionCommand::validate`.

## Notes For Agents
- **Realtime/safety:** none of this runs on the audio thread. `CommandHistory`, serialization, and coalescing allocate and may grow vectors — keep them strictly on the edit/MCP/UI threads. P05 mirrors validated manifest snapshots to the render path; do not add audio-callback coupling here.
- **Code pointers:** the brittle readers are `Command.cpp:64-151` (first-substring `json.find`); the over-broad transaction dispatch trigger `serializedCommand.find("\"commands\":[")` is at `Command.cpp:3780-3789`; the throwing missing-field path is `Command.cpp:66-68`; default no-op `redo()` is `Command.hpp:40-42`; `TransactionCommand::validate` re-serializes children and re-parses them (`Command.cpp:3521-3537`) — that path must be made non-throwing and ideally avoid the re-parse round-trip.
- **Gotchas:** the `*EditHistory` classes consume `std::variant` command aliases and `std::visit` (`EditTools.cpp:125`); promoting families to `ICommand` lets you delete the visitor and variants entirely. `MidiClipStore`/etc. `getOrCreate` semantics (`Command.hpp:360`, `:702`) must be preserved or folded into the manifest. Confirm no external callers break — current callers of `commandFromSerialized`/`CommandHistory` are confined to `libs/commands` and `EditTools` (none in `ApplicationSession` yet), so the public surface change is contained.
- **Maps to existing tasks:** this implements the deferred substance of `docs/tasks/006-undoable-command-layer.md` at production grade (006 lists undo/redo, registry, transactions, persistence as goals never fully met). It is a prerequisite the `026-036` set assumed but never owned: `028` (arrangement timeline UI), `030` (piano-roll/drums/pattern UI), `032` (mixer/automation UI), and `034` (production MCP integration) all assume one chronological command timeline. P03 establishes that timeline; no `026-036` task currently owns command-layer unification.
