# P26 Determinism, plugin-hosting, audio-correctness, and realtime-deadline verification suite + CLI productionization

> **Area:** tests-qa · **Severity:** blocker · **Effort:** L · **Phase:** Verification & Release Readiness

## Objective
Prove — with executable, CI-enforced tests — every guarantee the product rests on: byte-for-byte render/export determinism, real plugin hosting (process/state/RT-safety/crash-isolation), spectral correctness of time-stretch/pitch/warp, and realtime deadline compliance on professional-size sessions measured against the buffer period with real RSS/disk and tracked history. In the same pass, productionize `lamusica_cli` into an ID-agnostic, `--json`, render-controllable, schema/migrate-capable, preview+confirmation-token batch surface that mirrors MCP — replacing the current first-track-hardwired stdout-regex smoke harness. This is the verification substance of legacy task 035 plus the CLI productionization no 026-036 task owns; it must fail CI on any divergence rather than narrate success.

## Decisions To Make
- **Redistributable test plugin & licensing:** which plugin ships for the hosting test (e.g. a permissively licensed VST3/AUv3 SDK example, or a self-built in-repo plugin). Bundling third-party binaries must be license-audited (AGPL/GPL contamination risk for an open-source DAW); prefer a build-from-source minimal plugin under a compatible license over a vendored binary.
- **Determinism scope:** is byte-exactness required across block sizes *and* across plugin involvement, or only for the native graph? Floating-point summation order and any plugin's internal nondeterminism may force a "native render is byte-exact; plugin renders are tolerance-bounded" split — decide and document the contract.
- **Golden refresh policy:** how goldens are regenerated and reviewed when the engine legitimately changes output (explicit `--update-golden` gated behind code review vs. auto-regen).
- **Perf gate thresholds & hardware:** absolute p99/period budget vs. relative-to-baseline regression; which CI runner class is the reference machine, and whether perf gates are blocking or advisory on shared CI (soak tests are noisy on virtualized runners).
- **CLI surface vs. MCP parity:** how closely CLI verbs mirror MCP tool names/semantics and the confirmation-token format; whether `--json` reuses the MCP `escapeJson`/result schemas (`libs/mcp_bridge/src/EditTools.cpp:11`) to avoid a second JSON dialect.
- **P28 gating:** whether P26 ships the spectral suite as `xfail`-until-P28 or P26 is sequenced strictly after P28 lands.

## Compromises And Tradeoffs
- **Accepted:** spectral-correctness tests are deferred behind P28 — they are skipped (with an explicit reason) rather than deleted, so the gap (`tests-qa: Time-stretch/pitch-shift/warp untested`) stays visible until the DSP exists. Writing them now against the planning-only `Warp.hpp` math (no resampling/interpolation in `AudioGraph.cpp::sampleNodeValue`, `libs/audio/src/AudioGraph.cpp:67`, which reads integer source frames 1:1) would test nothing.
- **Accepted:** plugin renders may be tolerance-bounded rather than byte-exact if a chosen plugin is internally nondeterministic; the native-graph render stays byte-exact.
- **Rejected MVP shortcut:** keeping the stdout-regex smoke tests (`tests/CMakeLists.txt:247-560`) — they assert log strings, not audio/files, and pass even if the WAV is silent or wrong-length. Replaced with artifact assertions per `covers_gaps: Smoke tests assert brittle stdout regex`.
- **Rejected MVP shortcut:** keeping the string-size benchmark proxies (`Performance.cpp:400-471`) — they measure `std::string::size()`, not DSP/RSS/disk, and cannot detect a realtime deadline miss.
- **Rejected MVP shortcut:** leaving the CLI hardwired to the starter project. Generic id-addressable verbs are required so the CLI can drive arbitrary projects headlessly.
- **Risks later tasks must revisit:** perf-history baselines must be re-pinned when P22/P23 UIs or new DSP land; the headless GUI test is shallow until P19-P23 expose real bindings; SR-conversion correctness (P29) is out of scope here and needs its own determinism coverage.

## Acceptance Gates
- `cmake --build build && ctest --test-dir build -L determinism` — two independent renders are byte-identical; renders at two block sizes are byte-identical; journal-replay render equals the live-edit render; the committed golden matches and CI fails with a first-differing-sample report on any divergence.
- `ctest --test-dir build -L plugin-hosting` (under TSan, P06 job) — a real plugin processes audio (verified transform), state round-trips to byte-identical re-render, the RT delivery path reports zero allocation/lock/file-IO/JSON/MCP violations, and a deliberately crashing plugin leaves the host alive.
- `ctest --test-dir build -L perf` — p99 per-block wall time is strictly under the buffer period for a 32–64-track session, zero xruns over the soak, the scaling curve is recorded, RSS/disk are measured via OS APIs (not estimates), and results are appended to the tracked history keyed by `MachineContext`.
- `ctest --test-dir build -L audio-correctness` — with P28 present, FFT pitch/duration/transient-at-marker assertions pass; with P28 absent, the suite is reported skipped with a reason (never silently green).
- CLI gates: `lamusica_cli --help` prints usage and exits 0; `lamusica_cli query <arbitrary.lamusica> --json` emits versioned JSON that validates against the committed schema; `lamusica_cli edit <project> set-clip-gain <any-clip-id> -6 --dry-run` prints a preview + confirmation token and mutates nothing; the same with `--confirm <token>` applies; `render --format wav --bit-depth 24 --range 48000:96000 --stems` honors every flag and reports a **post-dither** peak; `schema` prints the version and `migrate` upgrades an older-schema fixture in place.
- Smoke gate: every former stdout-regex test now asserts a real artifact (golden hash / WAV PCM / JSON schema); headless `lamusica_daw` launch + binding test runs in CI without a display.

## Notes For Agents
