# P05 Validated audio-graph snapshot, precompiled noexcept render path, and true lock-free SPSC command queue

> **Area:** audio-engine · **Severity:** blocker · **Effort:** M · **Phase:** Phase 1 — Realtime engine foundation
> **Depends on:** P04 · **Blocks:** P06, P09, P11, P15, P18, P20, P25, P28

## Objective
Build the realtime-safe mutation spine the whole DAW rides on: move every graph validation and schedule-derivation step out of the per-block render path into an off-thread snapshot builder, leave a single `noexcept` callback that touches only precompiled integer schedules and preallocated buffers (no throw, alloc, lock, file-IO, logging, JSON, or per-block topological sort / `std::string` comparison), guarantee an invalid/cyclic/oversized graph can never reach the callback (it degrades to silence at swap time, never crashes), and replace the fake `RealtimeCommandQueue` with a genuine lock-free SPSC ring. The two divergent render implementations (`AudioGraph.cpp::renderGraph` and `AudioEngine.cpp::renderGraphBlock`) are unified onto one snapshot-consuming path so offline render and live render are bit-identical and there is exactly one place to make RT-safe. This is the production spine P03's commands and P24/P25's MCP edits mirror to the audio graph via validated double-buffered snapshots.

## Dependencies
- `P04 — Live CoreAudio device I/O on a realtime thread driving the engine callback` — supplies the actual device-thread callback that will invoke the `noexcept` render path; without it the RT contract is untestable against a real audio thread.

## Deliverables
- **New** `libs/audio/include/lamusica/audio/GraphSnapshot.hpp` + `libs/audio/src/GraphSnapshot.cpp`: an off-thread `buildSnapshot(const AudioGraph&) -> std::optional<GraphSnapshot>` (or `expected` with diagnostic) that runs ALL validation currently scattered across `AudioGraph.cpp::validateGraph` (lines 119-167: output-node existence, empty/duplicate ids, unknown-connection refs, recursive cycle detection via `visitCycle` lines 94-115) plus the bound checks currently thrown inline in `AudioEngine.cpp::renderGraphBlock` (lines 391-399: node count > `maxRealtimeGraphNodes=32`, connection count > `maxRealtimeGraphConnections=128`). On success it emits a flat, POD `GraphSnapshot` holding: fixed-capacity integer node schedule in topological order, integer `sourceIndices`/`destinationIndices` per connection, precomputed per-connection gain inputs, resolved `outputNodeIndex`, and a copy of the node parameters the render reads (kind, frequency, gain, pan, fades, note sequences, sample spans). No `std::string` keys, no `std::map`/`std::set`, no heap growth on the hot path.
- **Change** `libs/audio/src/AudioEngine.cpp::renderGraphBlock` (currently lines 387-548): drop the `throw std::runtime_error` checks at lines 392/395/399/403/408/420/452, drop the per-block `graphNodeIndex` linear scans (lines 406/417/418, helper at 114-121) and the per-block Kahn sort (lines 411-453), and make the signature `noexcept`. It consumes the active `GraphSnapshot` and only fills `realtimeGraphBuffers_` and the output span. Header change in `libs/audio/include/lamusica/audio/AudioEngine.hpp` line 120 to add `noexcept`.
- **Change** `libs/audio/src/AudioGraph.cpp::renderGraph` (lines 205-301): retire the `std::map<std::string,std::vector<float>> buffers` allocation (line 217), the `topologicalOrder` string-keyed sort (lines 169-203) and `validateGraph`+`throw` at the top (lines 207-210). Re-express offline render as `buildSnapshot` + the shared snapshot-consuming kernel, so `renderGraph` and `renderGraphBlock` call the same DSP core. `libs/audio/src/Bounce.cpp` line 117 caller updates to the unified path.
- **Change** `RealtimeCommandQueue` in `libs/audio/include/lamusica/audio/AudioEngine.hpp` (lines 74-89) and `libs/audio/src/AudioEngine.cpp` (push/pop/empty/full/size/clear lines 146-184): convert to a real SPSC ring with `std::atomic<std::size_t> head_/tail_` using acquire/release ordering. Remove the shared mutable `size_` written from both ends (current bug: `++size_`/`--size_` at lines 153/164 are read/written by both producer and consumer). `push`/`pop` stay `[[nodiscard]] noexcept` and allocation-free; overflow returns `false`; capacity becomes a power of two to allow index masking. `clear()` is consumer-side only.
- **New** double-buffered snapshot holder on `AudioEngine`: `std::array<GraphSnapshot, 2>` (or two preallocated slots) + `std::atomic<int> activeSnapshot_`. `publishSnapshot(GraphSnapshot)` is the off-thread writer (writes inactive slot, releases the index); the callback reads `activeSnapshot_.load(acquire)`. A null/invalid published graph maps to the silence snapshot.
- **New** microbenchmark + stress test: `tests/unit/bootstrap_tests.cpp` already exercises the queue (lines 88-119) and the graph compile path (lines 177-332); add a benchmark asserting per-block cost scales O(nodes+connections) and is invariant to node-name length (build two graphs with 2-char and 256-char ids, identical topology, assert near-equal per-block time), plus a malformed-graph stress loop (cycles, dangling connections, >32 nodes, missing output node) asserting `buildSnapshot` rejects them, the published snapshot stays silent, and the `noexcept` callback never throws. Wire a TSan target in `tests/CMakeLists.txt`.

## Decisions To Make
- **Snapshot ownership / lifetime under RT swap:** triple-buffer vs. double-buffer + atomic index vs. RCU-style retire. Double-buffer + `std::atomic<int>` index is simplest and bounded, but the writer must not reclaim the slot the callback may still be reading — decide the hand-back protocol (writer only ever touches the non-active slot; reclaim deferred to the next publish).
- **Snapshot representation of node payloads:** copy sample/note vectors into the snapshot (simple, but `buildSnapshot` allocates off-thread and the snapshot owns large buffers) vs. share immutable `shared_ptr<const>` sample blobs referenced by index (avoids copying audio data but adds atomic refcount traffic). For P05 favor copy-of-parameters + shared immutable sample storage; revisit when P09/P10 introduce streaming.
- **Fixed capacity vs. growable schedule:** keep the hard `maxRealtimeGraphNodes=32` / `maxRealtimeGraphConnections=128` caps (`AudioEngine.hpp` lines 133-134) as the snapshot's static array sizes, or make capacity a build-time template/config so P15 (busses/sends) and P12 (plugins) can raise it without touching the callback. Decide the cap policy now since it is baked into the POD layout.
- **SPSC queue capacity and overflow policy:** 64 (current) vs. larger; and whether producer overflow (`push` returns `false`) is silently dropped, coalesced, or surfaced to a UI/MCP error channel. Transport commands must never be silently lost — decide the back-pressure contract that P20/P25 will rely on.
- **Failure surfacing for rejected graphs:** the callback degrades to silence, but the off-thread `buildSnapshot` rejection must reach the author (command result in P03, MCP error in P24/P25). Decide the diagnostic type returned (`std::expected<GraphSnapshot, GraphError>`).

## Compromises And Tradeoffs
- **Accepted:** snapshot build does the heavy work (validation, topo sort, parameter copy) off the audio thread and may allocate; we accept its cost and a one-block publish latency in exchange for a strictly RT-safe callback. **Rejected MVP shortcut:** keeping validation/topology inside the callback "because it's already integer-indexed" — the correction confirms `renderGraphBlock` already uses integer schedules and preallocated buffers but *still* throws (lines 392/395/399/403/408/420/452) and recomputes the Kahn sort + linear `graphNodeIndex` scans every block, which violates the RT contract; do not conflate the bounded-but-throwing block path with a safe one.
- **Accepted:** unifying `renderGraph` (offline, `std::map`/string-keyed, lines 205-301) and `renderGraphBlock` onto one kernel means a one-time risk of changing offline render output; lock it down with golden-render regression before/after. **Rejected:** leaving two divergent paths — that is precisely how the offline and live engines silently drift.
- **Accepted:** for P05 the snapshot copies node parameters and shares sample blobs; this is not the final zero-copy design. **Risk later tasks must revisit:** P09 (real user content + instrument nodes), P12 (plugin nodes / RT parameter delivery), P15 (busses/sends), P18 (sample-accurate automation) all extend the snapshot payload and the per-block kernel — each must preserve the `noexcept`/no-alloc contract and may force raising the static caps. P28 (time-stretch) adds per-node DSP state that must live in or beside the snapshot without per-block allocation.
- **Risk:** the SPSC queue is single-producer/single-consumer by construction; if P25 ever introduces a second producer thread (MCP + UI both enqueueing) this invariant breaks — document it loudly so P20/P25 add a serializing front-end rather than a second writer.

## Acceptance Gates
- `cmake --build build --target lamusica-tests && ctest --test-dir build --output-on-failure` passes, including the new snapshot/queue/stress cases.
- **Callback RT contract:** `renderGraphBlock` is declared `noexcept` (compiler-enforced) and a code-grep / static check shows no `throw`, `new`/`malloc`, `std::map`/`std::set`/`std::string` construction, mutex, file-IO, or logging in its body or in `buildSnapshot`'s callee set reachable from the callback. Feeding it any snapshot (including the silence snapshot) never throws or allocates — verify under an allocation-trapping allocator in the test.
- **Build-time rejection:** a stress loop feeding malformed graphs (cycle, dangling connection, duplicate id, empty id, missing output node, 33 nodes, 129 connections) to `buildSnapshot` returns rejection for each; the active published snapshot remains the prior valid graph or silence; the callback is invoked thousands of times during the loop and never throws.
- **SPSC correctness (TSan):** the new TSan target runs a producer thread enqueueing N transport commands while a consumer thread drains via `pop()`; assert exactly N commands received, in FIFO order, none lost/duplicated/torn, and `==0` ThreadSanitizer reports. Re-confirms `tests/unit/bootstrap_tests.cpp` lines 88-119 against the atomic implementation; overflow `push` past capacity returns `false` without UB.
- **Per-block cost stability:** microbenchmark renders the same topology with 2-char vs. 256-char node ids and asserts per-block time differs by < a small tolerance (string length must not affect cost), and that doubling node/connection count roughly doubles (not squares) per-block cost.
- **Glitch-free swap:** while the device-thread callback (from P04) is running, call `publishSnapshot` repeatedly with valid graphs; assert no torn read (output never references a half-published schedule) and no audible discontinuity beyond an intended block boundary.
- **Render parity:** golden offline render via the unified `renderGraph`/`renderGraphOffline` path (`AudioEngine.cpp` lines 572-577, `Bounce.cpp` line 117) is bit-identical to the pre-refactor output for the existing graph-compile test fixtures (`bootstrap_tests.cpp` lines 177-332).

## Notes For Agents
- **Realtime constraints:** the only work allowed on the audio thread is reading `activeSnapshot_` (acquire load), draining the SPSC queue (`processRealtimeCommands`, `AudioEngine.cpp` lines 203-207), filling `realtimeGraphBuffers_`, and writing the output span. Everything else — validation, topo sort, snapshot construction, diagnostics — happens in `buildSnapshot`/`publishSnapshot` off-thread.
- **Code-grounded pointers:** throwing block path = `libs/audio/src/AudioEngine.cpp:387-548` (throws at 392/395/399/403/408/420/452; linear `graphNodeIndex` helper at 114-121; Kahn sort at 427-453; preallocated buffers at 130-131/195-196/455-461). Divergent offline path = `libs/audio/src/AudioGraph.cpp:205-301` (`validateGraph` 119-167, `visitCycle` 94-115, `topologicalOrder` 169-203, per-call `std::map` buffers at 217). Caps at `AudioEngine.hpp:133-134`.
- **Gotcha:** `renderGraphBlock` calls `processRealtimeCommands()` at line 389 *before* validation — after the refactor, command draining and snapshot read must both be RT-safe; do not reintroduce any throw between them. `selectDevice` (lines 186-197) reallocates `realtimeGraphBuffers_` and must never run concurrently with the callback — keep device reconfig on the off-thread/stopped path.
- **Snapshot is the GraphCompiler's output target:** `libs/session/src/GraphCompiler.cpp` (header `libs/session/include/lamusica/session/GraphCompiler.hpp`) produces the `AudioGraph` consumed here; P09 will compile real content into it, so keep `buildSnapshot` decoupled from session types.
