# P27 Release pipeline: universal signing, notarization/stapling, hardened runtime+entitlements, Info.plist, provenance, crash reporting, SBOM, and docs

> **Area:** build-ci-packaging · **Severity:** blocker · **Effort:** L · **Phase:** Phase 7 — Verification, performance, and release
> **Depends on:** P11, P25, P26 · **Blocks:** P32

## Objective
Make LaMusica a real, shippable macOS product: a Developer-ID-signed, notarized, stapled, hardened-runtime universal binary that a fresh user can launch from a DMG online and offline with no Gatekeeper override, plus a tag/dispatch CI release job that builds, signs inside-out, notarizes with `notarytool --wait`, staples, and **hard-gates** on `codesign --verify --strict --deep`, `spctl --assess`, and `stapler validate` before publishing the DMG, signed checksums, and SBOM. It also delivers operational substance the current docs only describe: archived dSYMs and a working crash reporter (DAW + daemon) with symbol upload, version/commit provenance stamped into every binary, aggregated third-party + AGPL source-availability notices, broadened supply-chain checks (all CMake files scanned + JUCE content checksum, pinned runner/Xcode), a real security-disclosure contact, and the user/contributor/release documentation. This is the production release gate, explicitly not an MVP packaging pass.

## Dependencies
- `P11 — JUCE plugin host module + crash-isolated out-of-process scanner with persistent cache` — introduces the plugin-host worker process and the JUCE `juce_audio_processors`/SDK link surface that must be signed inside-out (every nested helper executable/dylib) and covered by the universal-binary, entitlements, and dSYM archival work.
- `P25 — Live MCP-to-running-DAW bridge, OS daemon lifecycle, sandbox confinement, and addressable undo registry` — supplies the production daemon (`lamusica_mcpd`) lifecycle and sandbox model that the release job must sign, crash-report, and provenance-stamp alongside the app and CLI.
- `P26 — Determinism, plugin-hosting, audio-correctness, and realtime-deadline verification suite + CLI productionization` — the verification suite and productionized CLI that the release job runs as a precondition; the reproducible-build claim builds on its determinism gates.

## Deliverables
- **Wire the curated Info.plist (file to change).** `apps/daw/Info.plist.in` already carries the usage strings, `LSApplicationCategoryType public.app-category.music`, and `LSMinimumSystemVersion 14.0` (`apps/daw/Info.plist.in:1-31`) but is never referenced by the build — `apps/daw/CMakeLists.txt:1-5` calls `juce_add_gui_app(... BUNDLE_ID "dev.lamusica.app")` with no `JUCE_CUSTOM_PLIST`/`JUCE_MICROPHONE_PERMISSION_*` wiring, so the synthesized plist ships instead. Either pass the configured plist via `juce_add_gui_app(... JUCE_CUSTOM_PLIST ...)` or set JUCE's `MICROPHONE_PERMISSION_ENABLED`/`MICROPHONE_PERMISSION_TEXT`/`PLIST_TO_MERGE` options so usage strings, category, and min-OS reach `Contents/Info.plist`.
- **Reconcile the bundle identifier (files to change).** CMake declares `dev.lamusica.app` (`apps/daw/CMakeLists.txt:4`); `apps/daw/Info.plist.in:10-11` declares `dev.lamusica.daw`. Pick one canonical id, set it in both, and add a check (extend `cmake/VerifyPackage.cmake`) that asserts the shipped `Contents/Info.plist` `CFBundleIdentifier`, usage strings, and `LSMinimumSystemVersion` are present.
- **Commit entitlements + hardened runtime (new file `apps/daw/lamusica.entitlements`).** Author `com.apple.security.device.audio-input` (mic) and the Apple-Events entitlement matching `NSAppleEventsUsageDescription`; wire it via JUCE's `HARDENED_RUNTIME_ENABLED`/`HARDENED_RUNTIME_OPTIONS` + `--entitlements` at sign time. The codesign recipe in `docs/release/macos-distribution.md:57-73` already uses `--options runtime --timestamp` but does **not** pass `--entitlements` and is undriven by any script.
- **Universal binary (files to change / decision).** Set `CMAKE_OSX_ARCHITECTURES=arm64;x86_64` in a new `release-universal` preset in `CMakePresets.json` (current presets are single-config Unix Makefiles, `CMakePresets.json:14-23`), or record an explicit single-arch decision in `docs/release/macos-distribution.md`. Must cover JUCE and plugin SDKs from P11.
- **Release CI job (new file `.github/workflows/release.yml`).** `on: { push: { tags: ['v*'] }, workflow_dispatch: }`; steps: build release/universal, sign inside-out (`--options runtime --timestamp --entitlements`), build DMG (CPack already configures `DragNDrop`/`UDZO`, `CMakeLists.txt:98-100`), `xcrun notarytool submit --wait`, `xcrun stapler staple`, then **gate** on `codesign --verify --strict --deep`, `spctl --assess --type execute`, `stapler validate` (fail the job on any non-zero). The existing `.github/workflows/ci.yml` only builds debug/asan/release and uploads an unsigned TGZ (`.github/workflows/ci.yml:43-51`).
- **Signing/verify scripts (new files `scripts/sign-macos.sh`, `scripts/notarize-macos.sh`, `scripts/verify-signature.sh`).** Codify the manual recipes in `docs/release/macos-distribution.md:57-97` into idempotent, parameterized scripts the release job and the release checklist (`docs/release/release-checklist.md:29-34`) both call, so the sign/notarize/staple steps finally have scripted backing.
- **dSYM archival + crash reporter (new module + workflow change).** Generate dSYMs in release/universal builds; archive them as a release artifact. Integrate a crash reporter for the DAW and `lamusica_mcpd` (crashpad/breakpad or an async-signal-safe signal+backtrace handler) that runs **off the audio thread**, writes a minidump/backtrace, and uploads symbols. The daemon entrypoint is `apps/mcpd/src/main.cpp:11`; the DAW entrypoint is `apps/daw/src/main_juce.cpp`.
- **Version/commit provenance (files to change).** Generate a `version.hpp` (configure-time `git rev-parse`, dirty flag, build date) and stamp it into binaries. The CLI currently hardcodes `std::cout << "lamusica-cli 0.1.0\n";` (`tools/cli/src/main.cpp:169-171`) with no commit SHA, and `lamusica_mcpd` has **no** `--version` flag at all (`apps/mcpd/src/main.cpp:11-44`). Add `--version` to all three binaries (app via menu/`--version`, daemon, CLI) printing version + commit SHA + dirty + build date.
- **THIRD_PARTY/NOTICE + AGPL source availability (new files `THIRD_PARTY-NOTICES.md`, `NOTICE`).** Aggregate JUCE + plugin-SDK notices; the project is `AGPL-3.0-or-later` (`LICENSE:1-9`) so the corresponding-source offer/pointer must ship inside the package. Add these to the `install(FILES ...)` set (`CMakeLists.txt:84-85`) and to the `VerifyPackage.cmake` required-entries list (`cmake/VerifyPackage.cmake:27-45`).
- **SBOM + signed checksums (new file `scripts/sbom.sh`, workflow change).** Emit an SPDX or CycloneDX SBOM and `SHA256SUMS` for the DMG/artifacts; sign the checksums (codesign-detached or GPG) and publish them in the release job alongside the DMG.
- **Broaden supply-chain checks (file to change `cmake/CheckDependencyLock.cmake`).** Today it reads only two build files (`cmake/CheckDependencyLock.cmake:16-18`: root `CMakeLists.txt` + `CompilerWarnings.cmake`) for banned downloaders and string-matches the dependency doc. Recurse over **all** `CMakeLists.txt`/`*.cmake` under the repo and add a real JUCE content checksum (hash a manifest of the `LAMUSICA_JUCE_PATH` tree, compare to a pinned digest) so a tampered or wrong-commit JUCE checkout is rejected — beyond the existing git `rev-parse` check (`CMakeLists.txt:50-61`). - **Pin CI runner/Xcode + vuln scan (file to change `.github/workflows/ci.yml` + new `release.yml`).** Replace `runs-on: macos-latest` (`.github/workflows/ci.yml:9`) with a pinned image (e.g. `macos-14`) and a pinned Xcode via `xcode-select`/`maxim-lobanov/setup-xcode`; add a dependency vulnerability scan step (e.g. OSV/Grype against the SBOM).

## Decisions To Make
- **Bundle identifier canon:** `dev.lamusica.daw` vs `dev.lamusica.app`. The daemon already uses a separate label `com.lamusica.mcpd` (`apps/mcpd/src/main.cpp:14`) — decide the full id namespace (app, daemon, plugin-host helper from P11) and whether they share a Team-ID prefix for entitlement inheritance.
- **Universal vs single-arch:** ship `arm64;x86_64` (doubles build/notarize time, requires x86_64-capable JUCE/plugin-SDK builds) or ship arm64-only with a documented, dated rationale. Decide now because it changes the preset, dSYM, and notarization matrix.
- **Crash-reporter technology + privacy:** crashpad (heavier, mature minidump upload) vs breakpad vs a minimal async-signal-safe signal+backtrace writer. Must be off-audio-thread and decide the symbol-upload endpoint. Crash-reporting **consent/opt-in** is explicitly deferred to P32 (which depends on this task) — P27 ships the capture/symbolication plumbing; do not enable network upload without the P32 consent gate.
- **AGPL compliance shape:** since plugins are loaded at runtime, decide the corresponding-source offer wording and whether a written offer or a permanent source URL pointer ships in `NOTICE`; confirm no bundled third-party asset/SDK conflicts with AGPL redistribution (carry forward the redistribution-rights constraint from 036).
- **SBOM format + checksum signing:** SPDX vs CycloneDX, and codesign-detached vs GPG for `SHA256SUMS`. Decide whether the SBOM is generated from the CMake link graph or a curated manifest.
- **Secrets handling in CI:** how the Developer ID cert, notarytool keychain profile / App Store Connect API key, and symbol-upload token are stored (GitHub encrypted secrets) and which steps are skipped on fork PRs (no signing identity available).

## Compromises And Tradeoffs
- **Accepted:** the public CI on PRs stays unsigned (TGZ contents check only, `.github/workflows/ci.yml:43-51`); signing/notarization run only in the gated release job where secrets exist. Forks cannot run the release job.
- **Accepted:** if a universal binary is too costly, an arm64-only artifact is acceptable *only* with an explicit dated decision recorded in `docs/release/macos-distribution.md` — do not silently ship single-arch as today (`covers_gaps`: "Single-arch arm64-only artifact").
- **Rejected MVP shortcuts:** keeping the codesign/notarize steps as copy-paste docs with no scripted/CI backing (current state, `docs/release/release-checklist.md:29-34`); shipping the JUCE-synthesized plist instead of the curated one; leaving the `dev.lamusica.app`/`dev.lamusica.daw` mismatch; a `--version` that lies (`lamusica-cli 0.1.0` with no commit, `tools/cli/src/main.cpp:169-171`); the `.invalid` security contact; a crash "pipeline" that exists only in docs (`covers_gaps`: "Notarization/codesigning and crash-symbol upload exist only in docs").
- **Profiling tooling tradeoff:** the current profiling story is a bare `-g` flag behind `LAMUSICA_ENABLE_PROFILING` (`CMakeLists.txt:21`, preset `CMakePresets.json:33-41`). This task delivers dSYMs/symbolication; a full profiling-instrumentation harness (Instruments/perf integration) is acknowledged as out of scope and must be revisited by a later perf task.
- **Risk to revisit later:** the JUCE content checksum pins a tree hash, not a signed provenance attestation; auto-update/delta-update is explicitly deferred (full DMG redownload per `docs/release/macos-distribution.md:105-108`); telemetry/consent (P32) is the follow-on that must land before any crash upload is enabled by default.

## Acceptance Gates
- `cmake --preset release-universal && cmake --build --preset release-universal && ctest --preset release` succeed; `lipo -archs` on `LaMusica.app/Contents/MacOS/LaMusica`, `lamusica_mcpd`, and `lamusica_cli` shows the decided arch set (both arm64+x86_64, or arm64-only matching the documented decision).
- After the sign+notarize+staple scripts run: `codesign --verify --strict --deep --verbose=2 LaMusica.app` exits 0, `spctl --assess --type execute --verbose LaMusica.app` reports accepted, and `stapler validate <dmg>` passes — all three as **hard gates** in `.github/workflows/release.yml` (non-zero fails the job).
- A clean macOS user account (no developer tools) opens the notarized+stapled DMG, drags to `/Applications`, and launches LaMusica **online and offline** with no right-click/override; the first record attempt triggers the microphone TCC prompt (proves the mic entitlement + `NSMicrophoneUsageDescription` reach the bundle).
- `lamusica_cli --version`, `lamusica_mcpd --version`, and the app's version surface each print version + commit SHA + dirty flag + build date; a build from a dirty tree marks dirty; the SHA matches `git rev-parse HEAD`.
- An induced crash in the DAW and in `lamusica_mcpd` produces a symbolicated backtrace/minidump resolvable against the archived dSYMs (verify with `atos`/`llvm-symbolizer`); the handler is async-signal-safe and never runs on the audio callback.
- The release job publishes a signed DMG, `SHA256SUMS` (signature verifies), and an SBOM; re-running the build from the same commit produces byte-identical primary artifacts (reproducibility, leaning on P26 determinism) or documents the non-deterministic inputs (timestamps) that are stripped.
- `cmake -P cmake/CheckDependencyLock.cmake` passes on a clean tree, scans **all** CMake files, and **fails** when (a) a `FetchContent_Declare`/`vcpkg`/`ExternalProject_Add` is added to any nested CMake file, or (b) `LAMUSICA_JUCE_PATH` points at a checkout whose content checksum/commit differs from the pin.
- `cmake -DPACKAGE=... -P cmake/VerifyPackage.cmake` asserts the package contains `THIRD_PARTY-NOTICES.md`/`NOTICE` + AGPL source pointer, the curated `Info.plist` fields, and **no** `.invalid` string anywhere in shipped docs/metadata.
- CI uses a pinned runner image + pinned Xcode; the vuln-scan step runs against the SBOM and fails on a known-critical advisory (or records an allowlist with justification).

## Notes For Agents
- **Realtime safety:** the crash reporter's signal handler must be async-signal-safe — no `malloc`, locks, `std::cout`, JSON, or file-open on the failing thread beyond `write(2)`-to-fd primitives; symbolication and upload happen in a separate process/thread. It must never be installed on or invoked from the audio callback (the P04/P05 RT thread). Provenance/version stamping is configure-time only and never touches the audio path.
- **Inside-out signing:** sign nested helpers/dylibs first (notably the P11 crash-isolated plugin-host worker process and any embedded frameworks) then the `.app`, then build the DMG, then notarize the DMG. The existing recipe (`docs/release/macos-distribution.md:57-73`) signs the app/daemon/CLI but predates the plugin-host helper and omits `--entitlements`; update it.
- **Plist wiring gotcha:** `juce_add_gui_app` synthesizes `Info.plist`; the curated `apps/daw/Info.plist.in` is dead unless explicitly wired. Verify the *built* `LaMusica.app/Contents/Info.plist` (not the source `.in`) carries `CFBundleIdentifier`, `NSMicrophoneUsageDescription`, `LSApplicationCategoryType`, and `LSMinimumSystemVersion 14.0`. P19's note already flags that the shell relies on these plist keys.
- **CheckDependencyLock today** (`cmake/CheckDependencyLock.cmake:16-27`) only reads `CMakeLists.txt` + `CompilerWarnings.cmake`; nested files (`apps/daw/CMakeLists.txt`, `libs/*/CMakeLists.txt`, `tests/CMakeLists.txt`, `cmake/*.cmake`) are unscanned. Use `file(GLOB_RECURSE)` and skip `build/`, `external/`, `_CPack_Packages/`. The string-match list (`cmake/CheckDependencyLock.cmake:9-10`) must be updated if you add the checksum to `docs/developer/dependencies.md`.
- **`_CPack_Packages/` and `build/` in tree:** treat as stale artifacts, not source-of-truth (`docs/developer/dependencies.md:20`); do not let globs pick them up.
- **Maps to existing tasks docs:** this single release gate **implements `docs/tasks/036-release-hardening-signing-docs-examples.md`** (signing/notarization/docs/examples) and the build/packaging substance of the deferred `docs/tasks/025-packaging-signing-docs-release.md` and `docs/tasks/002-build-and-dependencies` (architecture 002), and **folds in the realtime-perf-security packaging/crash/profiling/supply-chain gaps** (no crash reporting, supply-chain limited to a single pinned JUCE commit, placeholder security disclosure, notarization/symbol-upload only in docs) into one gate. 036/025's loose "where applicable" notarization language is replaced by the hard CI gates above. Do **not** weaken those gates back to documentation-only steps.
