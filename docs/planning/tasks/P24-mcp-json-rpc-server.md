# P24 Standards-compliant MCP/JSON-RPC server with edit/render/orchestration tools and correct JSON

> **Area:** mcp · **Severity:** blocker · **Effort:** L · **Phase:** Phase 5 — MCP Protocol & Daemon Productionization
> **Depends on:** P03, P07, P09 · **Blocks:** P25

## Objective
Replace the bespoke whitespace line-grammar in `libs/mcp_bridge` with a real JSON-RPC 2.0 / MCP server (`initialize` → capabilities, `tools/list` → declared tool schemas, `tools/call` → dispatch with id correlation and structured error objects), and route the already-implemented edit/render/asset/orchestration handlers — which today have *zero* non-test callers — through that transport. Adopt a single correct, fully-escaping JSON implementation across the library (the current `escapeJson` escapes only `"` and `\`), enforce capability gating and confirmation-token flow for destructive ops at the protocol boundary with a positive allowlist, add a Unix-domain-socket transport sharing one dispatch/auth path with stdio, harden auth (CSPRNG token, constant-time comparison, no token over loggable stdout), and add a persistent hash-chained tamper-evident audit journal. This makes the daemon a real, agent-drivable surface — the prerequisite for P25's live bridge — not a stub.

## Decisions To Make
- **JSON implementation:** hand-roll the minimal `JsonValue` DOM (zero new deps, full control, fits the dependency-lock policy) versus vendoring a vetted single-header lib. Bundling implies a license/SBOM entry (feeds P27). Recommendation: hand-roll a small, fully-escaping serializer/parser to stay dependency-light and audit-friendly.
- **MCP fidelity scope:** how much of the MCP spec to implement now — `initialize`/`tools/list`/`tools/call` are mandatory; decide whether `notifications/*`, `ping`, progress notifications, and resource/prompt primitives are in P24 or deferred. Render progress maps naturally to MCP progress notifications — decide whether to ship that now or expose progress only via a poll verb.
- **Transport framing:** newline-delimited JSON vs. `Content-Length` header framing (LSP-style, what most MCP stdio clients expect). Pick one and document it; socket and stdio must share it.
- **Audit journal format & crypto:** hash function (SHA-256 needs a dependency or a vendored impl; a non-cryptographic chain is weaker) and storage (one JSONL file vs. the P08 binary sidecar registry). Decide whether the chain anchor is signed or merely hash-linked, and the retention/rotation policy.
- **Confirmation-token lifetime & binding:** the token from `previewCommand`/`renderConfirmationToken` (`EditTools.hpp:183`, `RenderTools.hpp:110`) must bind to the exact previewed payload and expire; decide TTL, single-use semantics, and whether it survives reconnection (interacts with `recoverConnection`, `DaemonSession.cpp:132`).
- **PluginControl capability:** activate it (gate `PluginEditCommand` apply behind it) or remove it from the enum. It is currently dead — pick one and reflect it in the capability matrix and `tools/list`.

## Compromises And Tradeoffs
- **Accepted:** P24 ships the protocol substance and durable audit, but P24 operates on the *offline* session/manifest snapshot, not a live running engine. Live mutation of a playing DAW, OS daemon lifecycle hardening, sandbox confinement (`sandbox-exec`/seatbelt), and an addressable cross-process undo registry are explicitly **P25**. The async render worker here runs against `RenderJobQueue` offline renders, not the realtime audio thread.
- **Rejected MVP shortcuts:** no "good-enough" `escapeJson` patch (control chars and UTF-8 must be correct everywhere — `acceptance: all produced JSON validates`); no keeping the line grammar with quoting hacks (it structurally cannot carry args with spaces/slashes/quotes); no name *denylist* as the security boundary (must be a positive allowlist of declared tools); no `mt19937` token (`DaemonSession.cpp:13`) and no `==` token comparison.
- **Risks later tasks must revisit:** confirmation-token semantics and the audit journal will need re-anchoring once P25 introduces a live engine and persistent daemon process (tokens crossing process restarts; journal continuity across daemon relaunch). The path sandbox must converge with P09's single asset-read guard and P27's hardened-runtime entitlements. If render progress is exposed only by polling now, P25 may need to add push notifications. Covers gaps: standards-compliant server, edit/render/orchestration tools over transport, persistent tamper-evident audit, socket transport, hardened auth, correct JSON, finer-grained scoping + PluginControl resolution, allowlist guard, and the realtime-perf-security gap "no destructive-op confirmation tokens/preview/undo-id plumbing in the MCP path".

## Acceptance Gates
- `cmake --build build --target lamusica_mcp_bridge lamusica-mcpd && ctest --test-dir build --output-on-failure -R 'mcp|jsonrpc|audit'` passes.
- A reference MCP client completes `initialize` → `tools/list` → `tools/call` over **both** stdio and the Unix-domain socket against a real spawned `lamusica-mcpd`; tool arguments containing spaces, slashes, and embedded quotes (e.g. a track name `"My / Bus \"A\""`) round-trip byte-exact through call params and JSON results.
- A destructive `tools/call` apply over the wire is **rejected** unless it carries the confirmation token returned by the matching preview; a read-only-scoped session is denied edit and render verbs; `PluginControl` is enforced for plugin edits or the capability is gone from `tools/list` (no dead capability).
- A render `tools/call` returns immediately with a job id, the daemon stays responsive to a concurrent query while the job runs, the job reports increasing progress, a cancel verb transitions it to `Cancelled`, and output paths `../escape.wav` and `/etc/passwd` are rejected by the sandbox.
- A fuzz/property test asserts every JSON the server emits parses back and contains no raw control chars and only valid UTF-8; tokens are ≥128-bit CSPRNG-derived, compared in constant time, and never appear in `--logs`/stdout/the audit journal in plaintext.
- The audit journal is append-only, contains every edit/render/destructive/denied op, survives a daemon restart (re-load verifies the hash chain), and a test that flips one byte of a past record makes chain verification fail.
- `grep -R "splitWords\|mt19937\|serializeProtocolResponse" libs/mcp_bridge/src` returns nothing in the production dispatch path (line grammar fully removed).
